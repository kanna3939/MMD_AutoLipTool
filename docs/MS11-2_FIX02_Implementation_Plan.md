- Project: MMD_AutoLipTool

## 1. 目的

MS11-2_FIX01 後も残っている、次の内容上の不整合を修正する。

- 発話区間内で、対象母音イベント自体は存在しているにもかかわらず、
  最終出力 VMD 上では **全母音モーフがほぼ 0 状態となり、瞬間的な開口だけが残る** 問題

本マイルストーンの目的は、**MS11-2 / FIX01 の基本仕様を維持したまま、writer 後段の正規化・保護・cleanup の相互作用により発生しているゼロ縮退を抑止すること** である。

## 3. 問題定義

### 3-1. 問題
発話区間に対応する母音イベントが存在しているにもかかわらず、最終出力 VMD において、

- イベント全体は消えていない
- しかし rise / fall の肩部分が消失し
- 上辺近傍または一部 frame のみ正値が残り
- 結果として **発話中なのにほぼ全母音ゼロで、一瞬だけ口が開く**

という状態が発生する。

### 3-2. 代表症状
ユーザー観測では、283 frame〜296 frame の `え` 区間が代表例であり、

- イベント自体は残存している
- ただし 0.0 キーが密に残り
- 一部の正値 frame だけが残る

という形で出力されている。

---

## 4. 現時点での原因整理

### 4-1. 主原因整理
現行 writer の後段ロジックでは、依然として以下の組み合わせが残っている可能性が高い。

1. 開口判定が **補間後の連続形状ではなく keyed frame 寄り**
2. MS11-2 / legacy fallback をまたぐ近接イベントで、**肩部分の保護が弱い**
3. suppression が正値肩部分を 0.0 に変換する
4. cleanup / zero prune が、元アンカー由来の 0.0 を残す
5. 結果として、**頂点近傍だけ残って肩が消える**

### 4-2. 問題の性質
この問題は、MS11-2 の shape 生成そのものより、**shape 生成後の正規化・保護・cleanup の組み合わせで発生する内容上の不整合** とみなす。

### 4-3. 現時点での修正方針上の含意
FIX02 では、次を狙う。

- 正常なイベント内部で rise / fall の寄与が不当に消されないこと
- 保護対象を「頂点」ではなく「発話イベントとして成立している区間」に寄せること
- cleanup が発話イベント内部の意味ある形状を潰さないこと

## 6. 必須要件

### R-01
発話イベントの `start_frame`〜`end_frame` 内において、**イベント自体が有効であるにもかかわらず、全母音が実質 0 のまま継続する状態** を、writer 後段で発生させないこと。

### R-02
MS11-2 の正規 4点台形イベントについて、上辺だけでなく **rise / fall の肩部分も含めて形状として保護できること。**

### R-03
legacy fallback を含む近接イベントについても、**発話イベント内部の有効な正値寄与** が short open / short pulse 系の判定で過剰に潰れないこと。

### R-04
cleanup / prune は、不要な中間 0.0 キーを整理しつつも、**発話イベント内部の有効形状を 0.0 列へ縮退させないこと。**

### R-05
開始ゼロ・終了ゼロ・衝突回避に必要なゼロは維持すること。

### R-06
4 frame 未満フォールバック仕様は維持すること。

### R-07
VMD バイナリ構造正常性を損なわないこと。

---

## 7. 挙動要件

### B-01
発話イベントが成立している区間では、最終出力上で **頂点近傍だけが残り、肩部分が全消失する** 挙動を抑止すること。

### B-02
発話イベント内部では、少なくともイベント形状として意味のある
- 立ち上がり
- 上辺
- 立ち下がり

の連続性が維持されること。

### B-03
発話イベント内部で、**全母音が 0.0 のまま連続する区間** が不自然に生成されないこと。

### B-04
開始ゼロ・終了ゼロ・衝突回避ゼロは、必要条件を満たす限り維持されること。

### B-05
無音区間非ゼロ除去に関する FIX01 の意図は維持すること。

---

## 8. 修正対象の考え方

### 8-1. 主修正観点
FIX02 では、特に次の3観点を補正対象とする。

1. **イベント内部保護の粒度不足**
2. **short open / short pulse 判定の過剰適用**
3. **cleanup / zero prune による肩部分消失**

### 8-2. 具体的な補正の方向
- 保護単位を「ピーク点」中心ではなく「発話イベント区間」寄りで扱う
- 近接イベントや疎キー列に対して、肩部分が short 判定で潰れにくい条件へ補正する
- cleanup が発話イベント内部のゼロアンカーを過剰保持して、結果として全体を閉口化しないよう補正する
