# MS9-2_Implementation_Plan.md

## 1. 文書の位置づけ

本書は、MS9-1 完了後に残った GUI 仕上げ課題を、**MS9-2** として切り分けて整理した実装計画・仕様補助文書である。

本段階で扱う対象は、主に以下に限定する。

- 起動直後の初期レイアウト調整
- 操作パネルの外観整理
- 入力部品のサイズ最適化
- アイコン資産の差し替え
- 余白・整列・表示密度の調整
- 最小ウィンドウサイズ制約の追加

本段階では、以下は対象外とする。

- shared viewport の責務再設計
- 再生制御仕様の変更
- 波形解析・VMD 出力ロジックの変更
- 設定永続化の本格実装
- 多言語化本実装

---

## 2. MS9-2 の目的

MS9-2 の目的は、MS9-1 までで成立した GUI 構造・表示同期・スクロールバー追加後の状態を前提として、  
**実画面確認で見えた外観上の違和感や、初期表示時のレイアウト不均衡を整理・調整すること**である。

特に以下を重点対象とする。

- 初期ウィンドウサイズと表示比率の適正化
- 左右カラム比率の是正
- 操作ボタンの幅広化抑制
- 数値入力欄の過剰横幅抑制
- assets アイコンへの差し替え
- ボタンラベルの改行整理
- GUI 全体の余白・サイズ感・見た目密度の統一

---

## 3. 固定前提条件

### 3-1. 初期ウィンドウ条件
- 初期ウィンドウ幅は **1270px** とする
- 初期表示の縦横比は **16:9 基準**とする
- 初期レイアウトは、標準的な横長モニター上で自然に見えることを優先する

### 3-2. 最小ウィンドウサイズ制約
- ウィンドウの最小サイズ制約を設ける
- 少なくとも **幅 720px 未満にはしない**
- この値は暫定値とし、後続マイルストーンで見直す可能性を残す
- ただし MS9-2 では、この暫定リミットを実装対象に含める

### 3-3. 左右カラム比率
- 中央コンテンツの左右比率は **左3 : 右7** とする
- 左側はテキスト・ファイル情報系表示領域
- 右側は波形表示・プレビュー表示領域
- この比率は **初期ウィンドウサイズ基準の初期値**として適用する
- リサイズ後まで常時固定することは本要件には含めない

### 3-4. フォント基準
- 基本フォントサイズは **11pt** とする
- フォントファミリーは **Windows 基準で自然に見える標準系**を前提とする
- 主要 UI 要素で過度なサイズ差が出ないようにする

対象例:
- 操作ボタン
- ラベル
- 入力欄
- パネル見出し
- 主要表示テキスト

### 3-5. アイコン基準
- アイコンは Qt 標準アイコンではなく、**assets フォルダ内画像**を使用する
- アイコンサイズは **Windows エクスプローラーの「中アイコン」相当**を基準とする
- 操作ボタン群で、過度なサイズ差が出ないように揃える

### 3-6. 余白基準
- GUI の余白・間隔は **4px～6px 程度**を基本基準とする
- 主な対象は以下とする
  - ボタン間隔
  - ラベルと入力欄の間隔
  - セクション内余白
  - パネル内パディング
  - アイコンと文字の距離

### 3-7. サイズ設計の解釈ルール
以下の具体値は厳密固定しない。

- ボタン最大幅の px 値
- 入力欄固定幅の px 値
- アイコン実表示 px 値
- 各余白の個別 px 値

ただし、これらはすべて次の解釈ルールで決定する。

- **中アイコンと文字が自然に収まること**
- **4～6px 程度の余白があること**
- **必要以上に横へ伸ばさないこと**
- **初期表示時に密度と視認性のバランスが取れていること**

## 4. MS9-2 実装対象

### 4-1. 初期ウィンドウ表示条件の調整
- 初期ウィンドウ幅を 1270px に設定する
- 初期縦横比を 16:9 基準で設定する
- 起動直後の表示バランスが横長レイアウトとして自然になるようにする

### 4-2. 最小ウィンドウサイズ制約の追加
- 最小サイズ制約を設定する
- 少なくとも幅 720px 未満にはならないようにする
- 必要に応じて高さ側も自然な範囲で制約するが、主条件は幅側とする

### 4-3. 左右カラム初期比率の修正
- 中央コンテンツの初期比率を左3 : 右7に設定する
- テキスト関連の左領域を狭める
- 波形・プレビューの右領域を広めに確保する

### 4-4. 操作ボタンの幅上限制御
- ウィンドウ横幅拡大に伴ってボタンが過度に横長化しないようにする
- ボタン幅は必要十分な大きさに留める
- 最大幅またはそれに準ずる制御で、余剰スペースをボタンへ無制限配分しないようにする

判断基準:
- 中アイコンが収まる
- 11pt 文字が収まる
- 4～6px 程度の内側余白が確保される
- それ以上の無駄な横拡張は避ける

### 4-5. 操作ボタン文言の改行整理
- 主操作系（TXT 読込 / WAV 読込）
- 処理実行系
について、ボタンラベルの改行を一旦なくす
- 1行表示前提で、アイコンと文字の収まりを再調整する
- EN 対応時の再評価可能性は残すが、本段階では日本語 1 行表記を優先する

### 4-6. モーフ最大値入力部のレイアウト修正
- モーフ最大値設定を、ラベルと入力欄の横並びに整理する
- 入力欄が横に過度に伸びないようにする
- 小数点以下 3 桁を含む表示が自然に収まる範囲を必要十分とする

判断基準:
- 11pt 基本フォントで視認できる
- 不要な横占有を避ける
- ラベルとの距離は 4～6px 程度を基準とする

### 4-7. 数値入力系 UI サイズの統一
- モーフ最大値入力欄を含む数値入力系 UI は、同じ設計方針でサイズ調整する
- ラベルと入力欄のバランスが崩れないようにする
- 必要以上に横へ広げないことを共通ルールとする

### 4-8. アイコン資産の差し替え
- Qt 標準アイコンを assets フォルダ内画像へ差し替える
- 対象は少なくとも以下を含む
  - TXT 読込
  - WAV 読込
  - 処理実行
  - VMD 保存
  - 再生
  - 停止
  - 拡大
  - 縮小
- 表示サイズは中アイコン相当を基準とする

### 4-9. 操作パネル内の余白・間隔調整
- ボタン間の余白を再調整する
- アイコンと文字の距離を揃える
- パネル内側余白を整理する
- 幅固定後に不自然な空きや詰まりが出ないようにする

### 4-10. 基本フォント適用条件の統一
- 操作ボタン、ラベル、入力欄、パネル見出しなどの主要 UI に 11pt 基準を反映する
- Windows 環境で視認性が不自然にならないことを優先する

### 4-11. 初期表示時の全体バランス確認前提の調整
- 起動直後状態で、以下が意図どおりに見えるように調整する
  - 左右領域の占有比
  - ボタン横幅
  - アイコンと文字の収まり
  - モーフ最大値入力欄の幅
  - 全体の余白密度
- 判定は「初期表示の自然さ」を優先する

## 5. 実装方針

### 5-1. 本段階の性質
MS9-2 は、新規機能マイルストーンではなく、**MS9-1 で成立した GUI を仕上げるための外観調整段階**として扱う。

### 5-2. 実装判断の優先順位
実装時は、以下の優先順位で判断する。

1. 機能成立を壊さないこと
2. 起動直後の見た目が自然であること
3. 必要以上に幅を消費しないこと
4. Windows 標準アプリとして違和感が少ないこと
5. モックアップ完全一致ではなく、実用性・可読性を優先すること

### 5-3. 詳細設計に委ねる事項
以下は本書で厳密固定せず、実装時の詳細設計に委ねる。

- ボタン最大幅の最終 px 値
- 入力欄固定幅の最終 px 値
- アイコン実表示 px 値
- 各レイアウト余白の個別 px 値
- 主要行高・ボタン高さの最終値

ただし、決定時は以下に従うこと。

- 中アイコン + 11pt 文字が自然に収まる
- 4～6px 程度の余白がある
- 無意味に横へ伸ばさない
- 初期表示全体としてバランスがよい

## 6. 対象外事項

MS9-2 では以下を行わない。

### 6-1. 設定永続化
以下のような保存処理は、当初予定どおり **MS10** で扱う。

- テーマ設定保存
- splitter 比率保存
- ウィンドウサイズ保存
- 言語設定保存
- その他 UI 状態保存

理由:
- GUI 仕上げ途中で永続化を入れると、未確定レイアウト状態が保存仕様へ先に固定されやすい
- まず MS9-2 で初期表示仕様と見た目を安定化させ、その後に MS10 で保存対象を確定する方が安全である

### 6-2. shared viewport / 再生制御の再設計
- スクロールバー責務の再設計
- Zoom / Pan 仕様の再変更
- 再生位置同期の設計変更
は対象外とする

### 6-3. 処理ロジック側の変更
- 波形解析ロジック
- 母音変換ロジック
- VMD 出力ロジック
- 音量解析ロジック
は対象外とする

---

## 7. 完了判定の目安

MS9-2 の完了は、少なくとも以下を満たした時点とする。

- 起動直後の初期サイズが 1270 幅・16:9 基準で自然に表示される
- 左右比率が左3 : 右7 の初期値として反映される
- ボタンが過度に横長にならない
- モーフ最大値入力欄が短幅・横並びで自然に表示される
- 操作ボタンの改行が解消される
- assets アイコンへ差し替わる
- GUI 全体の余白・密度が不自然でない
- 幅 720px 未満に縮小できない
