# MS11-2_FIX01_Implementation_Plan

- Project: MMD_AutoLipTool
- Target Milestone: MS11-2_FIX01
- Scope: writer 正規化整合修正
- Date: 2026-03-27

---

## 1. 目的

MS11-2 実装後に確認された、VMD 内容上の次の論理不整合を修正する。

1. 発話区間内で、モーフ値 `0.0` のキーが不自然に連続して並ぶ問題
2. 本来は閉口であるべき無音区間に、`0.0` を超えるモーフ値が残る問題

本マイルストーンの目的は、**VMD 構造の正常性を維持したまま、writer 内の正規化・抑制・最終出力整理の論理整合性を回復すること** である。

### 2-1. 完了前提
- MS10 は完了済み
- GUIFIX 系は完了済み
- MS11-1 は完了済み
- MS11-2 は完了済み

### 2-3. 今回の修正位置づけ
本対応は **MS11-2 のやり直しではなく、MS11-2 完了後の writer 正規化整合修正** である。

## 3. 問題定義

### 3-1. 問題A：発話区間内のゼロ連続
発話タイミング自体は概ね妥当であるにもかかわらず、最終出力 VMD 内において、モーフ値 `0.0` のキーが連続して並ぶ区間が存在する。

### 3-2. 問題B：無音区間への非ゼロ漏れ
本来は閉口であるべき無音区間にもかかわらず、最終出力 VMD 内に `0.0` を超えるモーフ値が残る区間が存在する。

---

## 4. 原因整理

### 4-1. 主原因
MS11-2 では 4点台形を疎なアンカー列として保持している一方、MS11-1 系の正規化判定が **補間後の実効形状ではなく keyed frame 単位** で短開口判定を行っている。

その結果、

- 本来は一つの連続開口として扱うべき形状が
- keyed frame 上では短い断片に見え
- short open / short pulse 系の抑制対象として誤検出され
- 正値キーが `0.0` に置換される

という流れが発生しうる。

### 4-2. 副次原因候補
- MS11-2 保護範囲が形状全体ではなく一部に偏っている可能性
- 抑制処理が「削除」ではなく「0 置換」を中心としていること
- ゼロ退避や最終整理の影響で、許容外 frame に非ゼロが残る可能性

---

## 5. 修正方針

### 5-1. 基本方針
今回の修正対象は **writer.py を主対象** とする。  
まずは VMD 書き出しロジック内部で論理整合性を回復することを優先する。

### 5-2. 層ごとの扱い
- `writer.py`: 主変更対象
- writer 系テスト: 主変更対象
- `pipeline.py`: writer だけで説明不能な場合のみ確認対象
- `text_processing.py`, `main_window.py`, GUI 層: 今回は原則対象外

### 5-3. 論理方針
以下の順で整合性を回復する。

1. 正規化判定基準を keyed frame 中心から、補間後の実効開口を扱える基準へ寄せる
2. MS11-2 由来形状の保護範囲を、形状全体に対してより妥当に扱えるよう見直す
3. suppression 後に不要な `0.0` キーが大量残存しないよう整理する
4. 最終段で許容外区間の非ゼロを除去する

## 6. 必須要件

### R-01
短開口判定および短パルス判定は、**keyed frame のみを根拠にせず、補間後の実効開口状態を反映可能な基準** へ修正すること。

### R-02
MS11-2 由来の正規な 4点台形形状は、MS11-1 系正規化によって不必要に破壊されないよう、**保護範囲を形状全体に対して再定義できる構造** にすること。

### R-03
suppression により抑制されたキーについては、**単純な `0.0` 置換だけで終えず、不要キー削除または冗長ゼロ整理を行うこと。**

### R-04
最終出力段階で、**無音区間またはイベント許容外区間に残る非ゼロキーを除去できること。**

### R-05
MS11-2 の既存仕様
- 非対称 4点
- 単一上辺
- 上辺中央基準
- 4 frame 未満フォールバック

は維持すること。

### R-06
VMD のバイナリ構造正常性を損なわないこと。

## 9. 実装方針詳細

### 9-1. 正規化判定の見直し
MS11-1 系の短開口 / 短パルス判定は、現状の keyed frame 偏重から、補間後の実効開口を扱える方向へ修正する。

ただし、本修正では必要以上の全面再設計は避け、writer 内で閉じる形を優先する。

### 9-2. MS11-2 保護範囲の見直し
現状の保護が peak 周辺や一部 frame に偏る場合、`start_frame`〜`end_frame`、または shape 全域を保護対象として扱えるよう整理する。

### 9-3. suppression 後処理の見直し
抑制ロジックにより値だけ `0.0` にされた不要キーが最終出力に大量残存しないよう、削除または prune の考え方を導入する。

### 9-4. 最終出力前の許容範囲整理
イベントの許容範囲外に残る非ゼロキーを除去する最終整理段を追加または再設計する。
