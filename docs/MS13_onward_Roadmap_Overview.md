# MS13 以降ロードマップ概要
## Ver0.4 系マイルストーン俯瞰文書

## 0. 文書目的

本書は、`MMD_AutoLipTool` の `MS13` 以降を対象に、  
**Ver0.4 系の全体マイルストーンを俯瞰し、個別マイルストーン文書へ落とし込みやすくするための上位整理文書**である。

本書の目的は次のとおり。

- `MS13` 以降の大枠順序を固定する
- 各マイルストーンの目的と責務境界を明示する
- 個別実装書へ分解しやすい粒度で、対象・非対象・完了条件を整理する
- Antigravity が個別実装計画を切る際の基準文書とする

本書は詳細実装手順書ではない。  
本書は、**Ver0.4 系全体の進行順と責務分割を固定するための親ロードマップ**として扱う。

---

## 1. 基準時点

- ベースライン:
  - `Ver 0.3.x`
- 前提:
  - 初回公開可能な到達点はすでに達成済み
  - `MS12` 以前は Qt / PySide6 系を前提に進んでいる
  - `MS13` 以降は、`wxPython` への GUI 基盤移行を起点とする
  - `MS13` 以降は `Ver0.4.x` 系として扱う
  - 実装作業は Antigravity を主作業者として進める

---

## 2. Ver0.4 系の主方針

Ver0.4 系では、単なる機能追加ではなく、まず GUI 基盤を差し替える。  
その後、差し替え後の基盤上で入力拡張・UX 改善・品質改善・多言語拡張を進める。

したがって、`MS13` 以降の進行原則は以下のとおり。

1. **GUI 基盤移行を最優先する**
2. **既存機能 parity を先に回復する**
3. **再生系 UX を新基盤に合わせて再構築する**
4. **入力拡張は GUI 安定後に載せる**
5. **品質改善は基盤と入力が固まった後に進める**
6. **多言語対応は最後段に置く**

---

## 4. 全体依存マップ

### 4.1 依存の主鎖

`MS13 -> MS14 -> MS15 -> MS16 -> MS17 -> MS18 -> MS19`

`MS20` は、厳密には `MS18` / `MS19` 完了必須ではないが、  
日本語側の基盤と入力導線が固まってから着手する方が安全である。

### 4.2 依存の考え方

- `MS13` が土台
- `MS14` が日常利用復旧
- `MS15` が再生 UX の再構築
- `MS16` と `MS17` が入力拡張
- `MS18` と `MS19` が品質改善
- `MS20` が多言語入口

## 6. 各マイルストーンで共通して持たせるべき観点

個別マイルストーン文書へ落とす際は、各 MS に対して最低限以下の見出しを持たせる。

### 6.1 推奨見出しテンプレート

- 目的
- 基準時点
- 対象ファイル
- 非対象
- 実装範囲
- 完了条件
- テスト観点
- リスク
- 保留事項

### 6.2 Antigravity 向けに明記すべきこと

- どこまで実装し、どこから先は次 MS に送るか
- 今回変更してよい責務境界
- 変更を避けるべき領域
- 不明点が出た場合に止まる条件
- 仮置きで進めてよい判断点

## 7. 先に固定しておくべき非機能条件

### 7.1 ライセンス
- 最終配布は MIT 前提
- 有償 OSS は避ける
- 導入依存は、配布時運用まで確認可能なものに限定する

### 7.2 開発環境
- Windows 11
- VS Code
- Python 3 系

### 7.3 実装方針
- `core` と `vmd_writer` の責務はできるだけ維持する
- GUI 差し替えを理由に core 意味論を不用意に変えない
- GUI 改修とアルゴリズム改修を同時に広げすぎない

## 8. 全体リスク

### 8.1 MS13〜MS15 が長くなるリスク
GUI 基盤移行と再生系再構築は、Ver0.4 全体の前半を占める重い工程である。

### 8.2 parity 不足のまま後段へ進むリスク
MS14 完了前に後段へ進むと、バグ原因の切り分けが困難になる。

### 8.3 入力拡張と品質改善の混線
MS16 / MS17 と MS18 / MS19 を同時並行に広げると、責務が散りやすい。

### 8.4 英語対応の過大化
MS20 は認識だけでなくマッピング設計が本体であり、予想より大きくなる可能性がある。

---

## 9. 要約

MS13 以降は、以下の順で進める。

1. `MS13`: wxPython 基盤移行
2. `MS14`: wx 版 parity 回復
3. `MS15`: 波形 / Preview / 再生系再構築
4. `MS16`: mp3 対応 + WAV 同時出力
5. `MS17`: 音声のみ入力対応
6. `MS18`: 同音母音谷スムージング改善
7. `MS19`: 発音単位スロープ設定
8. `MS20`: 英語対応基礎

この順序により、  
**基盤移行 -> parity 回復 -> UX 再構築 -> 入力拡張 -> 品質改善 -> 多言語化**  
という流れが固定される。

本書を `MS13` 以降の全体俯瞰文書とし、  
個別マイルストーン文書は本書の各節を基に切り出すものとする。
