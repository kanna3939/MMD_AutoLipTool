# MS8D-2 要件変更・仕様変更ドキュメント

- 作成日: 2026-03-21
- 対象プロジェクト: `MMD_AutoLipTool`
- 対象段階: `MS8D-2`
- 位置づけ: 既存 `MS8D` の改訂版。元の MS8D を直接上書きするのではなく、要件変更と仕様変更を整理した差分文書として扱う。

---

## 1. 目的

本書は、既存の `MS8D`（表示詳細化）を、

- 最小変更優先
- 表示詳細化の限定導入

という前提から、

- ユーザー操作性重視
- 可視性重視
- 編集ソフト寄りの表示操作導入

へ改訂した `MS8D-2` の要件変更と仕様変更を整理することを目的とする。

本書の適用範囲は GUI 表示層・表示操作層に限定する。内部データの正本、解析処理、VMD 出力ロジックの正本は変更しない。

---

## 2. 背景

既存文書上の `MS8D` は、以下を対象としていた。

- 横軸フレーム表記
- ズーム
- 同期ズーム基盤
- パス中間省略

ただし、これは「秒内部保持のままフレーム表記へ置換」「Zoom は WAV 読込後のみ有効」「波形と Preview Area の同期表示方針が成立する」という、比較的最小構成の表示改善として定義されていた。

一方、本改訂では、ユーザーインターフェース段階としての確認性・操作性を優先し、以下を追加で要求対象に含める。

- 汎用パン操作
- Preview Area 側のフレーム目盛り
- Zoom 操作のメニュー導線

これにより、`MS8D-2` は「表示詳細化」から一段進み、**共通ビューポートを持つタイムライン表示基盤の導入**として扱う。

---

## 3. MS8D から MS8D-2 への要件変更

### 3.1 変更方針

`MS8D-2` では、最小変更優先よりも、次を優先する。

1. 人間にとっての見やすさ
2. 人間にとっての操作しやすさ
3. 波形と Preview Area の表示一貫性
4. 既存責務分割の維持
5. 内部秒データ正本の維持

### 3.2 要件変更一覧

| 項目 | 既存 MS8D | MS8D-2 |
|---|---|---|
| Zoom | WAV 読込後のみ有効な基本ズーム | WAV 読込後のみ有効を維持しつつ、実用的な離散段階ズームを導入 |
| 表示範囲 | 同期ズーム基盤 | 同期ズームに加え、汎用パン操作を含む共通ビューポート基盤 |
| 波形横軸 | フレーム表記 | フレーム表記を維持し、主目盛りを 30fps 基準で扱う |
| Preview Area | 同期表示方針 | 同期表示に加え、フレーム目盛りを持つ表示対象へ拡張 |
| パス表示 | 中間省略 | 中間省略 + tooltip フルパス表示 |
| Zoom 導線 | ボタン中心 | ボタン + メニュー導線を同期対象とする |

### 3.3 MS8D-2 の新しい目的

`MS8D-2` の目的は、**秒ベース内部正本を維持したまま、波形表示と Preview Area に共通ビューポートを導入し、フレーム基準での視認・拡大・移動・比較を行える GUI を成立させること**とする。

---

## 4. MS8D-2 で固定する仕様

### 4.1 Zoom 仕様

- Zoom は離散段階の倍率方式とする。
- 例: `1x -> 2x -> 4x -> 8x`
- Zoom の有効条件は `WAV 読込後のみ` とする。
- Zoom In / Zoom Out の有効無効判定の正本は `main_window.py` とする。
- 操作パネルのボタンとメニュー action の有効無効は同期する。

### 4.2 ビューポート仕様

- 共有単位は秒ベース絶対値とする。
- `ViewSync` は、MS8C の共有秒位置に加え、共有可視範囲を扱う。
- 可視範囲は、少なくとも `start_sec` / `end_sec` を持つものとする。
- 波形表示と Preview Area は、同一の可視範囲を参照して描画する。

### 4.3 パン仕様

- `MS8D-2` では汎用パン操作を対象に含める。
- パンは再生位置変更ではなく、表示範囲変更として扱う。
- パン導入後は、Zoom 基準は「0秒固定」ではなく、**現在の表示範囲を基準として拡大縮小可能**な仕様へ拡張する。
- 0 秒はフレーム原点として維持するが、表示左端の固定点とはしない。

### 4.4 波形表示仕様

- `WaveformView` を波形表示の中心部品として維持する。
- 内部データは秒ベースのまま保持する。
- 横軸表示はフレーム表記とする。
- 主目盛りは 30fps 基準で扱う。
- 30fps 縦線表示は既存切替機能を維持する。
- 再生カーソル、母音ラベル、イベント範囲表示は、可視範囲に追従して再描画される。

### 4.5 Preview Area 仕様

- Preview Area は波形とは別ウィジェットのままとする。
- 5段固定（あ / い / う / え / お）は維持する。
- `timeline` / 整形済み表示データのみを見る方針を維持する。
- 描画は全体正規化固定ではなく、共有可視範囲に基づくクリップ描画へ拡張する。
- Preview Area にもフレーム目盛りを持たせる。
- Preview Area のフレーム原点は波形表示と一致させる。

### 4.6 パス表示仕様

- TEXT / WAV のパス欄は basename 表示ではなく、中間省略表示へ変更する。
- 省略形式は `先頭ディレクトリ + ... + ファイル名` を基本とする。
- 例: `C:\...\voice.wav`
- ラベル表示は省略形、tooltip はフルパスとする。
- 主読込・自動補完の見せ方は既存方針どおり区別しない。

### 4.7 メニュー仕様

- Zoom In / Zoom Out は操作パネルのボタンだけでなく、メニュー action としても提供する。
- ボタンとメニュー action は同一状態を反映する。
- 既存の View メニュー表示切替（30fps 縦線 / 母音ラベル / イベント範囲 / 表示初期化）との整合を保つ。

## 5. 責務分割の改訂

### 5.1 `main_window.py`

持たせる:
- Zoom / Pan / 表示初期化の司令塔責務
- ボタン / メニュー action の状態判定
- `ViewSync` への共有可視範囲更新
- WAV 読込成功 / 失敗 / 無効化時の表示範囲初期化
- パス表示文字列と tooltip の反映

持たせない:
- 波形描画ロジック本体
- Preview 重整形
- `timeline` 直接加工
- `pipeline.py` / `writer.py` の既存ロジック変更

### 5.2 `view_sync.py`

持たせる:
- 共有秒位置の保持・配布
- 共有可視範囲の保持・配布
- 表示範囲初期化通知

持たせない:
- 描画そのもの
- WAV 再生制御本体
- フレーム文字列整形の UI ロジック

### 5.3 `waveform_view.py`

持たせる:
- 波形描画
- フレーム軸表示
- 可視範囲に基づく表示更新
- パン / ズーム結果に基づく再描画
- 既存 overlay と再生カーソルの可視範囲対応

持たせない:
- 共有可視範囲の正本管理
- `timeline` 正本管理
- Preview 表示責務

### 5.4 `preview_area.py`

持たせる:
- 5段固定描画
- 整形済み Preview データの受け取り
- 共有可視範囲ベースのクリップ描画
- フレーム目盛り描画
- 再生カーソル描画

持たせない:
- `timeline` 直接解釈
- 重い整形
- 状態判定の正本
- 再生制御本体

### 5.5 `preview_transform.py`

持たせる:
- `timeline -> PreviewData` 整形責務

持たせない:
- Zoom / Pan 状態
- 可視範囲依存処理
- GUI 描画ロジック

`preview_transform.py` は原則非変更対象とする。表示範囲や目盛りは `preview_area.py` 側で扱う。

## 6. MS8D-2 の対象範囲

### 6.1 対象

- 波形横軸のフレーム表記強化
- 共有可視範囲を前提とした Zoom
- 共有可視範囲を前提とした Pan
- 波形 / Preview Area の同期ビューポート
- Preview Area のフレーム目盛り
- Zoom ボタン / メニュー導線の同期
- パス中間省略 + tooltip フルパス

### 6.2 非対象

以下は `MS8D-2` でも非対象とする。

- `pipeline.py` の挙動変更
- `writer.py` の挙動変更
- `whisper_timing.py` の挙動変更
- Preview 用整形契約の変更
- 再生位置の手動シーク機能そのもの
- 波形クリックによる再生位置ジャンプ
- VMD 出力仕様変更

注: 本書でいう Pan は表示範囲の移動であり、再生ヘッドのシーク機能とは区別する。

## 7. 完了条件（MS8D-2）

以下を満たしたとき、`MS8D-2` 完了とみなす。

1. 波形表示が秒内部保持のままフレーム表記で確認できる。
2. Zoom In / Zoom Out が WAV 読込後のみ有効である。
3. Zoom 操作がボタンとメニューの両方から行え、状態が同期する。
4. 波形表示と Preview Area が同一可視範囲で表示される。
5. 汎用パン操作により可視範囲を移動できる。
6. 再生カーソルが可視範囲内で整合して表示される。
7. Preview Area にフレーム目盛りが表示される。
8. 30fps 縦線表示の既存切替が維持される。
9. TEXT / WAV パス欄が中間省略表示となり、tooltip でフルパスを確認できる。
10. 既存主要導線（TEXT 読込 / WAV 読込 / 処理実行 / VMD 出力）を崩さない。
11. 内部データ正本は秒ベース絶対値のまま維持される。
12. `preview_transform.py` / `pipeline.py` / `writer.py` / `whisper_timing.py` の責務を広げない。

## 8. 既存文書に対する差分上の注意

### 8.1 `repo_milestone.md` に対する差分

既存 `MS8D` 行は、最小構成の表示詳細化を前提としている。
本書の `MS8D-2` は、以下を追加した拡張版である。

- 汎用パン操作
- Preview Area フレーム目盛り
- Zoom メニュー導線

### 8.2 `Specification_Prompt_v3.md` に対する差分

既存仕様書では、

- 波形のフレーム表記
- Zoom の有効条件
- パス中間省略
- ボタンとメニューの有効無効同期

は既に整合する。

一方、以下は本書で新たに固定する追加仕様である。

- Preview Area 側のフレーム目盛り
- 汎用パン操作
- 可視範囲共有を前提とした `ViewSync` 拡張
- `0秒固定ズーム` ではなく `現在表示範囲基準ズーム` への拡張

### 8.3 `MS8C_Implementation_Handoff.md` / `Version_Control.md` に対する差分

MS8C 文書では、Zoom / 手動シーク / 表示詳細化は未着手のまま維持されている。
本書は、その未着手範囲のうち GUI 表示詳細化を `MS8D-2` として再定義するものである。

## 9. 実装判断上の固定メモ

- 秒ベース正本は維持する。
- フレーム値は表示・補助計算用の派生値とする。
- パンは表示移動であり、再生位置シークとは分ける。
- Preview Area の拡張は描画責務の範囲で完結させ、整形責務へ戻さない。
- `main_window.py` は引き続き司令塔とする。
- `OperationPanel` は表示・signal 送出に留める。
