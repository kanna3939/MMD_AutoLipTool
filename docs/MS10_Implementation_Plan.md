# MS10_Implementation_Plan

## 1. 目的

MS10 は、現行 GUI 基盤の上に **設定永続化** と **多言語化本実装** を導入するマイルストーンとする。  
MS9 / MS9-2 で整備した GUI・表示層を前提に、以下を成立させることを目的とする。

- 日本語 / 英語の実行中切替
- 言語設定の次回起動時復元
- 履歴の次回起動時復元
- 重要 UI 設定の永続化
- 文字列定義の正本一元化
- 設定破損時・保存失敗時の安全動作

---

## 2. MS10 のスコープ

### 2-1. 対象に含むもの

- 設定永続化機構の新規導入
- `settings_store.py` の新規作成
- `.ini` 形式での設定保存 / 復元
- 言語設定の保存 / 復元
- 履歴設定の保存 / 復元
- 重要 UI 設定の保存 / 復元
- `i18n_strings.py` を正本とした文字列集約
- `main_window.py` / `waveform_view.py` を含む直書き文字列の回収
- 実行中の言語切替 UI 実装
- 起動時の OS 言語判定
- 設定破損時フォールバック
- 保存失敗時通知

### 2-2. 対象に含まないもの

- 波形表示オプションの永続化
- 多言語 JSON 外部化
- 3 言語以上への拡張
- UI デザインの再調整
- 波形描画ロジックや再生ロジック自体の改修
- 出力仕様や音声解析仕様の変更

---

## 3. 設定永続化方針

## 3-1. 保存対象

MS10 で正式に永続化対象とするのは、次の最小構成に限定する。

### 重要設定（都度保存）
- テーマ設定
- splitter 比率
- ウィンドウサイズ
- 言語設定

### 終了時保存対象
- 履歴の各項目
  - 最近使った TEXT ファイル履歴
  - 最近使った WAV ファイル履歴

## 3-2. 保存対象外

次の項目は MS10 では永続化しない。

- 波形表示オプション
  - フレームグリッド ON/OFF
  - 母音ラベル ON/OFF
  - イベント区間表示 ON/OFF
- その他、一時的表示状態
- 再生状態
- 一時選択状態
- 描画内部状態

## 3-3. 保存形式

- 設定ファイル形式は `.ini`
- 新規作成ファイル名は **`MMD_AutoLipTool.ini`**

## 3-4. 保存先

- 保存先は **exe ルート**
- 開発実行時も同じ方針に準拠し、アプリ実行基準位置から扱う

---

## 4. 起動時復元方針

起動時の設定復元は、次の順序で適用する。

1. ウィンドウサイズ
2. splitter
3. 言語
4. UI 構築
5. 文言反映
6. テーマ

この順序を正とし、実装中に変更しない。

---

## 5. 多言語化方針

## 5-1. 対応言語

- 日本語
- 英語

## 5-2. 言語切替 UI

- **メニュー方式** とする

## 5-3. 初期言語決定ルール

初回起動時、または保存済み言語設定が取得できない場合は、OS 言語設定を読む。  
読めない場合の優先順は次の通りとする。

1. 英語
2. 日本語

## 5-4. 多言語化対象範囲

MS10 では、次を一括して多言語化対象とする。

- メニュー
- ファイルダイアログ
- 警告文
- 状態文
- 空状態文言
- 軸ラベル
- ツールチップ
- バージョン情報

対象範囲から意図的に除外する文言は設けない。  
ソース上の直書き文字列は原則として回収対象とする。

## 6. `i18n_strings.py` 運用ルール

## 6-1. 基本方針

- `i18n_strings.py` を文字列定義の正本とする
- 多言語文字列は外部 JSON へ出さない
- Python 内部管理を継続する

## 6-2. 構成方針

- **画面責務ごとのクラス分割** を維持する
- 各クラス内は **用途別の大文字定数名** で統一する

例:
- `MainWindowStrings`
- `OperationPanelStrings`
- `StatusPanelStrings`
- `WaveformViewStrings`
- `CommonStrings`

## 6-3. 共通文言

- OK / Cancel / Error など、複数箇所で共通利用する文言は共通領域へ集約する

## 6-4. フォーマット文言

- フォーマット文言はテンプレート文字列として定義する
- 埋め込み値の組み立ては呼び出し側で行う

## 6-5. 文言作成方針

- 文言の厳密固定は行わない
- 一般的に理解できる内容であれば、実装時の文言は AI 判断に委ねる

---

## 7. `settings_store.py` 方針

## 7-1. 新規作成

- `settings_store.py` は MS10 で新規作成する

## 7-2. 主責務

- `.ini` 読み込み
- `.ini` 書き込み
- 既定値生成
- 値検証
- 不正値フォールバック
- 設定破損時の安全復旧
- 保存失敗時の状態管理

## 7-3. `main_window.py` との役割分担

### `settings_store.py`
- 保存 / 読込 / 値検証 / フォールバック

### `main_window.py`
- 現在 UI 状態の取得
- UI への設定適用
- 設定変更契機での保存呼び出し
- 終了時の履歴保存呼び出し
- 言語切替反映

---

## 8. 安全動作方針

## 8-1. 設定破損時

- 設定ファイルが壊れていても起動不能にしない
- 破損時は安全なデフォルト値で起動する
- 不正値は既定値へフォールバックする

## 8-2. 保存失敗時

保存失敗時は、次の方針とする。

- **初回失敗時のみ** 警告ダイアログを 1 回表示する
- 以後、そのセッション中の保存処理は無効化する
- 画面上の変更はそのセッション中は反映してよい
- ただし、次回起動時には保持されないことを利用者へ伝える
- 詳細な例外全文は既定表示しない
- 文言は要約中心とする

---

## 9. 履歴管理方針

- 永続化後も、現行と同じ件数上限を維持する
- 無効な履歴項目は起動時に除去する
- 履歴保存は終了時に行う
- 履歴保存失敗時もアプリ本体機能は継続利用可能とする

## 11. 実装上の基本ルール

- 既存の GUI 責務分離を崩さない
- 既存の出力仕様・解析仕様を変更しない
- 多言語化は文字列差し替えに限定し、UI 再設計へ広げない
- 永続化は最小対象に留める
- 新規文言は一般的に理解できる範囲で AI 判断可
- 仕様未確定事項を増やさない

---

## 12. 完了条件

MS10 の完了条件は次の通りとする。

### 12-1. 言語切替
- 日本語 / 英語を実行中に切替できる
- メニュー、ダイアログ、警告文、状態文、空状態文言、軸ラベル、ツールチップ、バージョン情報が切り替わる

### 12-2. 永続化
- テーマ設定が次回起動で復元される
- splitter 比率が次回起動で復元される
- ウィンドウサイズが次回起動で復元される
- 言語設定が次回起動で復元される
- 履歴が次回起動で復元される

### 12-3. 安全動作
- 設定ファイル破損時でも起動できる
- 不正値は既定値へフォールバックされる
- 保存失敗時、初回のみ警告ダイアログが出る
- 保存失敗後もアプリ本体は継続利用できる

### 12-4. 文字列管理
- 対象範囲の直書き文字列が `i18n_strings.py` へ集約される
- 多言語化漏れがない

---

## 13. 実装判断メモ

- 多言語文字列の正本は Python 内に維持する
- 外部 JSON 化は採用しない
- 波形表示オプションの永続化は行わない
- 文言の細部は一般的に理解できる範囲で AI 判断に委ねる
- 本計画書に記載された決定事項を MS10 実装時の正とする
