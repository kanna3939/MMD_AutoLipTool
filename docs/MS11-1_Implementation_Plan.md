# MS11-1_Implementation_Plan

## 1. 目的

MS11-1 は、現行の VMD モーフ出力基盤の上に、**重複キーフレーム正規化** と **瞬間開閉の緩和** を導入するマイルストーンとする。  
MS10 までで整備した GUI・設定永続化・多言語化基盤を前提に、今回は **VMD 出力結果の不整形抑制** に対象を限定する。

本マイルストーンで達成したいことは、次のとおりとする。

- 同一モーフ・同一フレーム重複を最終 VMD 出力から除去する
- 母音全体の開口状態を踏まえ、単一モーフだけを見た誤判定を避ける
- `0 → 中間値 → 0` 型の孤立短開口を緩和する
- 既存の開始ゼロ / 終了ゼロ保証を維持する
- 既存の通常台形出力を極力維持したまま、不自然な短パルスのみを抑制する

## 3. 問題定義

現行出力では、次の問題が発生し得る。

### 3-1. 同一モーフ・同一フレーム重複

同一モーフ・同一フレームに、複数のキーフレームが残る場合がある。  
これには次のようなパターンが含まれる。

- `0.0` と非ゼロ値の共存
- 非ゼロ値どうしの重複
- `0.0` どうしの重複

これは最終 VMD 出力として不整形であり、明示的に正規化対象とする。

### 3-2. 瞬間開閉

時間キーフレームが隣接している場合、特に 3 フレーム程度の短い塊で

- `0.0 → 中間値 → 0.0`

のような形が発生し、一瞬だけ口が開いてすぐ閉じる見え方になることがある。  
これは現行 writer の短区間フォールバックでも自然に起こり得る。

### 3-3. 単一モーフだけ見た誤判定

あるモーフ単体が `0 → 中間 → 0` でも、同フレーム帯で他モーフが非ゼロなら、口全体としては閉口していない。  
したがって、瞬間開閉の判定は **単一モーフ単位ではなく、母音全体の開口状態** を基準に行う必要がある。

## 4. 実装方針

### 4-1. 基本方針

MS11-1 では、形状表現を増やすのではなく、**既存出力の不整形を減らす** ことに集中する。  
そのため、既存の timeline 生成ロジックは極力維持し、writer 側の最終出力直前に **後処理型の正規化層** を追加する。

### 4-2. 実装位置

主実装位置は `writer.py` とする。  
責務整理は次のままとする。

#### `pipeline.py`
- 母音 timeline の生成
- 区間情報の構築
- RMS 等を踏まえた peak 値の決定

#### `writer.py`
- timeline から VMD モーフフレームへの展開
- 最終モーフフレーム正規化
- VMD 書き出し

### 4-3. 実装順

MS11-1 の処理順は、次を正とする。

1. 既存どおり morph frame を生成
2. 既存の同一フレーム衝突回避を適用
3. 同一モーフ・同一フレーム重複を統合
4. 母音全体の開口状態を構築
5. 孤立短開口を検出
6. 孤立短開口を緩和
7. 必要最小限の個別モーフ短パルスを整理
8. 最終ソートして VMD 書き出しへ渡す

この順序は固定とし、途中で入れ替えない。

## 5. 重複正規化方針

### 5-1. キー識別単位

重複判定の識別単位は、次とする。

- `(morph_name, frame_no)`

### 5-2. 統合ルール

同一 `(morph_name, frame_no)` に複数キーがある場合、1 件に統合する。  
統合ルールは次のとおりとする。

- 非ゼロ値が 1 件でもあれば、非ゼロ値を優先する
- 非ゼロ値が複数ある場合は、最大値を採用する
- すべて `0.0` の場合は、`0.0` を 1 件だけ残す

### 5-3. 既存処理との関係

既存の「rise-start の 0 を `frame-1` にずらす」処理は、当面残してよい。  
ただし、その後に **一般正規化を必ず通す** こととする。

---

## 6. 母音全体の開口判定方針

### 6-1. 開口判定の必要性

瞬間開閉の判定を単一モーフだけで行うと、他モーフが開口しているケースを誤って閉口扱いする危険がある。  
そのため、MS11-1 では **母音全体の開口状態** を内部的に構築する。

### 6-2. 内部表現

フレーム単位で、次の値を取得できる内部表現を持つ。

- あ
- い
- う
- え
- お
- 総合開口量 `open_value`

### 6-3. 総合開口量

初回実装では、総合開口量を次で定義する。

- `open_value(frame) = max(あ, い, う, え, お)`

### 6-4. 閉口 / 開口判定

微小値を吸収するため、閾値 `epsilon` を導入する。

- `open_value <= epsilon` を閉口
- `open_value > epsilon` を開口

この判定は、瞬間開閉の検出専用とする。

---

## 7. 孤立短開口の検出方針

### 7-1. 基本方針

MS11-1 で対象とするのは、**本当に孤立した短い開口区間** のみとする。  
単純に「3 フレームだから削除」のような固定長判定は採用しない。

### 7-2. 緩和対象条件

次を満たす開口区間のみを、孤立短開口として扱う。

- 前後が閉口
- 開口区間が短い
- 母音全体で見て孤立している
- ピーク値が極端に高くない

### 7-3. 誤判定防止

あるモーフ単体が `0 → 中間 → 0` でも、同フレーム帯で他モーフが非ゼロであれば、口全体として閉口ではない。  
この場合、孤立短開口として扱わない。

---

## 8. 孤立短開口の緩和方針

### 8-1. 緩和方式

MS11-1 では、変形台形や再構成は行わず、**削除 / 0 化による抑制** に限定する。

### 8-2. 緩和の優先順位

優先順位は次のとおりとする。

1. まず母音全体の連続性を壊さないこと
2. 次に不自然な一瞬開閉を減らすこと
3. 最後に個別モーフの細かいチラつきを減らすこと

### 8-3. 維持すべきもの

- 既存の開始ゼロ / 終了ゼロ保証
- 通常の台形出力の基本構造
- 短区間フォールバックの大枠

---

## 9. 個別モーフ短パルス整理方針

### 9-1. 扱い

母音全体では開口が続いているが、特定モーフだけが極短で立って消えるケースは、補助的に整理してよい。  
ただし、これは MS11-1 の主目的ではない。

### 9-2. 整理条件

次を満たす場合のみ、削減候補とする。

- そのモーフ区間が極短
- そのモーフを除去しても母音全体の開口 / 閉口列が変わらない
- 前後の主導モーフ遷移を壊さない

### 9-3. 注意

ここで過剰に整理すると、後の変形台形や多ポイント台形の拡張余地を削る可能性がある。  
そのため、MS11-1 では攻めすぎない。

---

## 10. 安全動作方針

### 10-1. 既存仕様維持

MS11-1 では、不整形抑制のために次を壊してはならない。

- 通常台形出力
- 短区間フォールバック
- 開始ゼロ / 終了ゼロ
- 書き出し順の安定性
- 最大フレーム数など既存ガード

### 10-2. 丸め由来衝突への配慮

秒単位では別イベントでも、30fps 丸めで同一フレームへ潰れることがある。  
このため、最終正規化は **秒単位ではなく最終フレーム番号ベース** で行う。

### 10-3. 二重管理回避

既存の衝突回避と新規正規化が二重管理にならないよう注意する。  
ただし、MS11-1 では大規模リファクタリングは行わず、まずは最小限の安全な積み増しを優先する。

## 13. テスト方針

### 13-1. 重複統合テスト

最低限、次を確認する。

- `0.0` と非ゼロが同一フレームに共存する場合
- 非ゼロ値が複数ある場合
- `0.0` が複数ある場合

### 13-2. 孤立短開口テスト

最低限、次を確認する。

- `0 → 中間 → 0` の孤立短開口が抑制される
- 他モーフが非ゼロのとき閉口扱いしない
- 短いが必要な開口区間は残る

### 13-3. 丸め衝突テスト

最低限、次を確認する。

- 秒単位では別でも、30fps 丸めで同一フレームに潰れたケースが正規化される

### 13-4. 既存仕様維持テスト

必要に応じて、次を再確認する。

- 開始ゼロ / 終了ゼロが維持される
- 通常台形出力が大きく崩れない
- 既存ガード条件が維持される

## 14. 実装上の基本ルール

- 既存の責務分離を崩さない
- 出力仕様全体を再設計しない
- 変形台形や多ポイント台形へ広げない
- writer を主変更対象に限定する
- 単一モーフだけで瞬間開閉判定をしない
- 判定は最終フレーム番号ベースで行う
- 仕様未確定事項を増やさない
