# MS15_Block_Breakdown

## 1. 文書情報

- 文書名: `MS15_Block_Breakdown.md`
- 対象プロジェクト: `MMD_AutoLipTool`
- 対象系列: `Ver 0.4.x`
- 対象マイルストーン: `MS15`
- 主対象読者: `Antigravity`
- 記述言語: `日本語`
- 目的: `MS15 を block 単位へ分解し、責務境界・依存関係・完了条件・流入禁止範囲を明確化する`

---

## 2. この文書の目的

本書は、MS15 を安全に実装するために、MS15 全体を block 単位へ分解し、各 block が何を担当し、何を担当しないかを固定するための文書である。

MS15 の主目的は、**MS14 で回復した wx 主系の実用コア導線の上に、波形 / Preview / 再生系の可視確認 UX を再構築すること** にある。  
したがって、MS15 は単なる音再生追加の段ではなく、**再生位置が見え、波形と Preview が同期し、必要最小限の追従表示まで成立する段** として扱う。

この目的を崩さないため、MS15 は以下の 6 block に分解する。

- MS15-B1: 波形表示基盤
- MS15-B2: Preview 表示基盤
- MS15-B3: 再生基盤と再生位置同期
- MS15-B4: オートスクロールと再生中追従 UX
- MS15-B5: 表示・再生状態統合と action state 整理
- MS15-B6: 統合整理と MS15 closeout

---

## 3. MS15 の全体位置づけ

### 3.1 前段との関係
MS14 により、以下は成立済みである。

- wx 主系起動入口
- TEXT 読込
- WAV 読込
- 解析実行
- VMD 保存
- recent files / last output dir / settings save
- busy / cancel / timeout warning / late callback discard
- 波形 / Preview / 再生の placeholder / disabled 整理

MS15 は、この基盤の上に、**視覚確認と再生確認のための表示・同期レイヤを追加する段** である。

### 3.2 後段との関係
MS16 以降では、mp3 対応、音声のみ入力対応、品質改善、英語対応などを扱う。  
そのため MS15 では、**表示・再生 UX の再構築に責務を限定**する。

### 3.3 MS15 の重要認識
- MS14 で core workflow は成立済み
- 現時点の waveform / Preview は placeholder 段階
- 再生操作は UI 上に置き場があるが、実動は未成立
- MS15 は「表示と再生確認」を成立させる段であり、入力形式拡張や品質改善を扱う段ではない

---

## 4. MS15 の固定方針

### 4.1 再生の最小成立範囲
MS15 における「再生」の最小成立範囲は、以下とする。

- WAV の再生開始
- WAV の再生停止
- 再生位置の取得
- 再生位置の波形 / Preview への同期反映

一時停止、任意シーク、ループ再生などは MS15 必須には含めない。

### 4.2 Zoom / Pan の扱い
MS15 では、Zoom / Pan は本実装対象に含めない。  
ただし、将来の拡張を壊さないように、**表示責務や viewport 境界を乱さない構造整理**は許容する。

### 4.3 オートスクロールの基準
MS15 のオートスクロールは、**再生位置が可視範囲の右端付近へ寄ったときに追従する方式**を基本とする。  
常時中央固定スクロールは初期方針に含めない。

### 4.4 Preview の責務
MS15 の Preview は、以下を主対象とする。

- 再生位置に応じた現在状態の可視化
- 時系列の母音イベント / 区間の可視化

ただし、これは**確認用可視化**であり、編集 UI や大規模インタラクション UI にはしない。

### 4.5 Qt 凍結側の扱い
Qt 側は、**仕様参照元としてのみ使用**する。  
wx 側への構造丸移植や責務丸写しは行わない。

### 4.6 block 分割粒度
MS15 は、**中粒度（5〜6 block 程度）**で分割する。  
1 block に複数責務を過度に抱え込ませず、かつ過分割にもならない粒度を維持する。

### 4.7 本文書で使う重要語
#### timing plan
`timing plan` は、MS14 で解析成功時に `current_timing_plan` として保持される、**母音イベント列・区間・時刻情報を含む解析結果**を指す。  
MS15 では、主に Preview 表示と再生位置同期の参照元として用いる。

#### MS14 で残された再生関連 stub
MS14 時点での再生関連 stub は、少なくとも以下を指す。

- UI 上の `Play` / `Stop` ボタンの置き場
- controller 側の `request_playback_play()` / `request_playback_stop()` の stub
- 再生操作を disabled または status 文言レベルに留める仮導線

MS15 では、これらを**実再生と同期導線へ置換**する。

#### playback state
本文書における `playback state` は、**再生の事実そのものを表す実行時状態**と、**それを UI / action state / invalidate へどう反映するかというアプリ状態解釈**を区別して扱う。  
この区別は B3 と B5 の責務境界で使用する。

## 5. MS15 の到達目標

MS15 全体の到達目標は、以下とする。

1. wx 主系で波形が placeholder ではなく実表示される
2. wx 主系で Preview が placeholder ではなく実表示される
3. WAV の再生開始 / 停止が wx 側から行える
4. 再生位置が波形 / Preview に同期反映される
5. 再生位置追従の最小オートスクロールが成立する
6. 再生中の action state / busy とは別の再生状態管理が最低限整合する
7. MS14 の core workflow を壊していない
8. Zoom / Pan を未実装のまま成立させる境界が維持されている
9. Preview は確認用可視化として意味を持つ
10. MS16 以降の内容を流入させていない

---

## 6. MS15 全体の設計原則

### 6.1 確認 UX 優先
MS15 では、見栄えよりも、**処理結果を確認できること**を優先する。

### 6.2 波形と Preview を分離しつつ同期させる
波形と Preview は別責務として実装する。  
ただし、再生位置・表示更新の基準は可能な限り共有し、視覚的不整合を避ける。

### 6.3 再生は「音が鳴る」だけで終わらせない
MS15 の再生は、**再生位置が見えること**までを含めて成立とする。

### 6.4 Qt 側の丸移植をしない
Qt 側は仕様参照元として扱う。  
wx 側では、wx に適した責務分割で再構築する。

### 6.5 Zoom / Pan を先食いしない
MS15 では表示・同期・追従を優先し、Zoom / Pan 本体は後続へ送る。

### 6.6 block 単位で閉じる
各 block は、後続 block の責務を前倒しで取り込まない。

| MS15-B2 | Preview 表示基盤 | 母音イベント / 区間と現在状態を Preview として可視化する |

| MS15-B6 | 統合整理と MS15 closeout | MS15 全体を統合し、MS16 へ送る境界を整理して閉じる |

## 8. 各 block の詳細

### 8.1 MS15-B1 波形表示基盤

#### 目的
MS14 の waveform placeholder を、wx 側での実波形表示へ置き換える。

#### 主対象
- WAV 解析結果を元にした波形描画
- 描画領域の最小座標系
- 可視範囲の最小概念
- 再生位置線の受け口
- 将来の viewport 拡張を壊さない最小構造

#### In Scope
- 波形の静的表示
- 横軸上の時間対応
- 再生位置マーカー受け口
- 最小再描画導線
- 表示更新のための view helper
- placeholder の置換

#### Out of Scope
- 音再生そのもの
- Preview 描画
- オートスクロール
- Zoom / Pan
- 編集操作
- 高度な描画最適化

#### `analysis_result_valid` との境界
B1 の波形表示は、**WAV 読込と WAV 解析結果に依存し、`analysis_result_valid` には依存しない**。  
すなわち、解析未実行または再解析前でも、WAV が有効なら波形表示自体は成立してよい。

#### 完了条件
1. WAV 読込後に波形が実表示される
2. `0.0 sec` と `duration_sec` の両端を基準に、任意の時刻を x 座標へ一意に写像できる時間軸規約が存在する
3. 再生位置線を後続 block から反映できる
4. placeholder 文言が不要になる
5. B3/B4 がこの波形表示基盤の上で進められる

---

### 8.2 MS15-B2 Preview 表示基盤

#### 目的
MS14 の Preview placeholder を、確認用の Preview 可視化へ置き換える。

#### 主対象
- 母音イベント / 区間の可視化
- 現在位置に応じた状態表示
- timing plan を元にした最小 Preview 表現
- 再生位置同期の受け口

#### In Scope
- 区間ベースまたはイベントベースの可視化
- 現在位置ハイライト
- 母音確認に必要な最小ラベル / 表示
- placeholder の置換
- 後続 block からの再生位置更新受け口

#### Out of Scope
- 編集 UI
- 直接ドラッグ編集
- Zoom / Pan
- 複雑なアニメーション
- VMD 編集機能

#### `analysis_result_valid` との境界
B2 の Preview 表示は、**有効な `timing plan` を前提とする**。  
したがって、`analysis_result_valid == False` の間は Preview 本表示を成立させず、未表示または非有効状態でよい。  
B5 はこの有効 / 無効の切替規則を UI 状態へ統合するが、B2 自体は「有効な解析結果があるときに何を描くか」を担当する。

#### 完了条件
1. 解析結果があると Preview が実表示される
2. 母音イベント / 区間が確認できる
3. 現在位置ハイライトの受け口がある
4. placeholder 文言が不要になる
5. B3/B4 がこの Preview 表示基盤の上で進められる

---

### 8.3 MS15-B3 再生基盤と再生位置同期

#### 目的
WAV の再生開始 / 停止を wx 側で実動化し、再生位置を波形 / Preview へ同期反映できる状態にする。

#### 主対象
- playback start
- playback stop
- 再生位置取得
- 再生位置更新ループ
- 波形 / Preview への同期通知

#### In Scope
- Play / Stop 実動化
- **実行時 playback state の最小導入**  
  例: `is_playing`, `playback_position_sec`, `playback_started_at` に準ずる最低限の再生事実管理
- 再生位置更新タイミングの最小設計
- 再生位置の波形反映
- 再生位置の Preview 反映
- controller / frame / view helper の最小接続

#### Out of Scope
- Pause
- 任意シーク
- ループ
- オートスクロール
- Zoom / Pan
- 再生中 GUI 滑らか化の最適化

#### B5 との責務境界
B3 は、**再生の事実そのもの**を成立させる block である。  
すなわち、

- 再生しているか
- 今どの位置か
- 表示へどう通知するか

を扱う。

一方 B5 は、B3 で導入された実行時 playback state を受けて、

- Play / Stop ボタンをどう enable/disable するか
- 再解析や再読込時にどう invalidate するか
- status 文言をどう整合させるか

といった**アプリ状態としての解釈と統合**を担当する。  
B5 は新たな再生事実ソースを作る block ではない。

#### 完了条件
1. WAV を再生開始できる
2. 停止できる
3. 再生位置が取得される
4. 再生位置が波形 / Preview に反映される
5. MS14 で残されていた再生関連 stub が、最小実装導線へ置換される

---

### 8.4 MS15-B4 オートスクロールと再生中追従 UX

#### 目的
再生確認性を上げるため、再生位置が可視範囲右端付近へ来たときの追従表示を導入する。

#### 主対象
- 可視範囲右端基準の追従
- オートスクロール発火条件
- 再生位置と表示範囲の整合
- 波形 / Preview の追従基準合わせ

#### In Scope
- 右端接近での追従ロジック
- 必要最小限の表示範囲更新
- 波形と Preview の追従整合
- 過剰スクロール抑制の最小調整
- **ネイティブスクロール操作が存在する場合に限り、ユーザーが連続スクロール / ドラッグ中は自動追従を一時抑止する最小競合回避**
- 将来の viewport 拡張を壊さない境界維持

#### Out of Scope
- 常時中央固定スクロール
- Zoom / Pan
- 手動スクロールと高度共存制御
- 再生体感の大規模最適化
- DAW レベルの挙動再現

#### 完了条件
1. 再生位置が可視範囲右端付近へ来ると追従する
2. 追従は再生位置が可視範囲内に十分収まっている間は不要に発火しない
3. 波形 / Preview の追従更新後も、同一再生位置に対して時間的に矛盾しない表示位置を保つ
4. Zoom / Pan を要求せずに追従仕様が成立している
5. 手動スクロール UI が存在する場合でも、連続手動操作中に自動追従が干渉し続けない最小抑止がある

---

### 8.5 MS15-B5 表示・再生状態統合と action state 整理

#### 目的
表示系と再生系の state / enable-disable / invalidate / status を、MS14 の core workflow を壊さない形で整理する。

#### 主対象
- 再生状態の UI 統合
- 表示更新状態
- 解析結果変更時の invalidate
- 再生可能条件
- Play / Stop ボタン状態
- status 更新の最小整合

#### In Scope
- **B3 の実行時 playback state を UI / action state へ統合する規則**
- `analysis_result_valid` と Preview 表示可能条件の関係整理
- WAV の有無と波形表示可能条件の関係整理
- 再読込 / 再解析時の表示 invalidate
- action state 再評価
- busy 中と playback 中の衝突回避ルール
- status 表示の最小整理

#### Out of Scope
- settings 永続化の大幅拡張
- 再生履歴
- 複雑な state machine 全面刷新
- i18n
- 高度なセッション復元

#### B1 / B2 との責務境界
- B1 は、WAV が有効なら波形を描ける状態を作る
- B2 は、解析結果が有効なら Preview を描ける状態を作る
- B5 は、その前提をもとに  
  - いつ表示を無効化するか  
  - 再解析要求時にどこを消すか  
  - Play / Stop をどの条件で有効化するか  
  を定義する

B5 は B1/B2 の描画ロジックを作る block ではなく、**それらをアプリ状態として接続する block** である。

#### 完了条件
1. 再生可能条件が定義されている
2. Play / Stop の enable-disable が最低限整合する
3. 入力変更 / 再解析時の表示 invalidate が壊れない
4. MS14 の分析 / 保存導線と衝突しない
5. B6 で統合確認できる状態になる

---

#### 主対象

- 波形 / Preview / 再生 / 同期 / 追従の結合確認
- 最小手動導線確認
- 軽量統合チェック
- README / docs の現在地同期
- MS16 非流入確認

#### In Scope
- MS15 全体の結合確認
- 接着不良の最小修正
- 軽量統合テストの追加または更新
- README / docs の現在地同期
- MS16 以降へ送る事項整理

#### Out of Scope
- mp3 対応
- 音声のみ入力対応
- 品質改善ロジック
- 英語対応
- Zoom / Pan 本実装
- 大規模リファクタ

#### 完了条件
1. B1〜B5 が統合されている
2. 波形 / Preview / 再生 / 同期 / 追従の最小導線が通る
3. MS14 の core workflow を壊していない
4. README / docs の現在地が一致している
5. MS16 へ安全に引き継げる

## 9. block 間の依存関係

### 9.1 基本依存
- B1 は先頭 block
- B2 は B1 と並行し得るが、**共通の時間軸規約**を前提に持つ
- B3 は B1 / B2 に依存
- B4 は B1 / B2 / B3 に依存
- B5 は B1〜B4 に依存
- B6 は B1〜B5 全体に依存

### 9.2 依存の意味
- B1 は波形表示の基盤を作る
- B2 は Preview 表示の基盤を作る
- B3 は再生と同期を乗せる
- B4 は追従 UX を乗せる
- B5 は状態遷移と操作可能条件を整える
- B6 は全体を closeout する

### 9.3 B1 と B2 の前提差
- B1 は、WAV 解析結果と時間軸規約を前提に進められる
- B2 は、時間軸規約に加えて valid な `timing plan` を前提に進める

したがって、B1 と B2 は同じ前提を持つわけではなく、**時間軸だけを共有し、解析結果依存は B2 の方が強い**。

---

## 10. flow 概要

MS15 全体の流れは、以下とする。

1. **B1** で波形の実表示基盤を作る
2. **B2** で Preview の実表示基盤を作る
3. **B3** で再生開始 / 停止と再生位置同期を実動化する
4. **B4** で右端接近基準のオートスクロールを導入する
5. **B5** で表示・再生状態と action state を整理する
6. **B6** で統合し、MS15 を closeout する

## 13. 主要リスク

### Risk 1. 表示と再生の責務混線
波形 / Preview / playback が 1 箇所に密結合する危険がある。  
**Control:** B1〜B5 の責務分割を厳守する。

### Risk 2. Qt 側の構造流入
Qt 側の構造をそのまま持ち込み、wx 側で不自然な責務になる危険がある。  
**Control:** Qt 側は仕様参照元としてのみ扱う。

### Risk 3. Zoom / Pan の先食い
表示改善の名目で viewport 拡張が流入し、MS15 が肥大化する危険がある。  
**Control:** Zoom / Pan 本体は明示的に対象外とする。

### Risk 4. 再生だけ成立して可視確認が弱い
音が鳴るだけで再生位置確認が弱いと、MS15 の価値が低下する。  
**Control:** 再生位置同期を再生成立条件に含める。

### Risk 5. オートスクロールが過剰になる
追従頻度が高すぎると体感が悪化する。  
**Control:** 右端接近基準を採用し、常時中央固定は避ける。

### Risk 6. MS14 core workflow 破壊
再生系追加で分析 / 保存導線が壊れる危険がある。  
**Control:** B5/B6 で state と action state を再点検する。

## 14. MS15 の Done 判定

MS15 は、以下をすべて満たしたら完了とみなす。

1. 波形が wx 側で実表示される
2. Preview が wx 側で実表示される
3. WAV の再生開始 / 停止ができる
4. 再生位置が波形 / Preview に同期する
5. 右端接近基準のオートスクロールが成立する
6. playback state / action state / invalidate が最低限整合する
7. MS14 core workflow を壊していない
8. Zoom / Pan 用のユーザー操作や完了前提を追加せずに MS15 が完結している
9. Preview が確認用可視化として成立している
10. MS16 以降の内容を流入させていない
11. README / docs の現在地が一致している

## 15. Antigravity への実装指針

- まず MS15 を「確認 UX 再構築」として扱うこと
- 波形 / Preview / 再生 / 追従を分けて実装すること
- 再生は start/stop + position sync までを最小成立範囲とすること
- B3 では再生事実を作り、B5 では UI / action state へ統合すること
- オートスクロールは右端接近基準で扱うこと
- Zoom / Pan を先食いしないこと
- Qt 側は仕様参照元としてのみ使うこと
- Preview は確認用可視化に留めること
- MS14 の分析 / 保存導線を壊さないこと
- MS16 以降の内容を流入させないこと
- B6 で docs / closeout まで閉じること

## 16. まとめ

MS15 は、Ver 0.4 系における **視覚確認・再生確認 UX の再構築段** である。  
MS14 が「最低限使える wx 主系」を成立させたのに対し、MS15 は「結果を見て、再生して、確認できる wx 主系」を成立させる段である。

そのため、MS15 では次を守ることが重要である。

- 波形と Preview を placeholder から実表示へ置き換える
- 再生は start/stop だけでなく position sync まで成立させる
- B3 と B5 の責務を混線させない
- 追従表示は右端接近基準で最小導入する
- Zoom / Pan は先送りする
- Qt 側は仕様参照元としてのみ扱う
- MS14 core workflow を壊さない

この block breakdown を、MS15 実装の正本方針として扱う。
