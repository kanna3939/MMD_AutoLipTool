# MS9_GUI_Requirements

## 1. 文書の目的

本書は、MMD_AutoLipTool における MS9「GUI整備」の要件を定義する詳細要件書である。  
MS9 では、既存の処理導線を維持したまま、GUI 全体の視認性・操作理解性・レイアウト柔軟性を向上させることを目的とする。

本書は GUI 層・表示層の整備を対象とし、音素解析・タイミング推定・VMD 出力仕様などの処理ロジック正本には踏み込まない。

加えて、MS9 では GUI 外観改善だけでなく、`main_window.py` の責務整理と GUI 構造の分散化を行い、後続の MS10 以降に耐えられる画面構成基盤を整えることも目的に含む。

本書は MS9 詳細要件の正本として扱う。  
全体仕様との整合確認は `Specification_Prompt_v3.md` を参照するが、MS9 の詳細責務・構造分散・GUI 要件の解釈は本書を優先する。

---

## 2. 目的

MS9 の目的は以下とする。

- モックアップを参照しつつ、レイアウトと主要外観を改善する
- Windows 向けツールとして自然で、視認しやすい GUI に整備する
- 後続の多言語化・設定永続化を見据え、文字長差や状態保持に耐えられる GUI 基盤を整える
- 実装最小化よりも、ユーザーフレンドリーな画面設計を優先する
- 既存の主要導線（TEXT読込 / WAV読込 / 処理実行 / VMD出力）を維持する
- `main_window.py` への責務集中を軽減し、表示系と接続補助系を適切に分散する
- MS9 の時点で、テーマ切替・左右分割比率・文字列管理の「受け皿」を用意する

## 3. スコープ

### 3-1. 対象

MS9 の対象は以下とする。

- 上部操作ボタン群の外観変更
- ボタン配置、折り返し、サイズ統一ルール整備
- 左右レイアウトの可変分割導入
- 左側テキスト領域の幅再調整
- 左側情報領域の独立パネル化
- 右側表示領域の独立コンテナ化
- 左右分割コンテナの導入
- セクション見出しと余白構成の整理
- Preview 行ラベル帯の視認性改善
- 波形表示エリア / Preview エリアの背景・配色見直し
- 波形表示エリアの不要表記削除
- ステータスバー表示構成の整理
- テーマ切替導線の追加
- 分析終了通知音の追加
- 多言語化を見据えた文言長耐性の整理
- 最小ウィンドウサイズ制約の見直し
- GUI 文字列管理の受け皿整理
- 左右分割比率保持の受け皿整理
- `main_window.py` の責務整理と GUI 系ファイル分散

### 3-2. 対象外

MS9 では以下を対象外とする。

- 音素解析・タイミング推定・VMD 出力仕様の変更
- `current_timing_plan.timeline` 等の正本データ構造変更
- Preview 正本責務の変更
- 再生制御仕様そのものの変更
- 多言語文字列定義機構の本格導入
- 設定保存機構そのものの本格実装  
  ただし、後続で永続化対象となる GUI 状態項目の整理と、受け皿追加は対象に含む
- `i18n_strings.py` / `settings_store.py` の全面導入  
  ただし、テーマ切替・左右分割比率・文字列管理の受け皿の置き場整理は対象に含む
- アイコン画像資産の最終確定  
  ただし、Qt 標準アイコンまたは暫定アイコンによる実装は対象に含む

## 4. GUI全体方針

- GUI整備では、モックアップを参照しつつも、「Windows 標準寄りの自然な見た目」を優先する
- GUI整備では、モックアップに対して「レイアウトと主要外観を寄せる」方針を採る
- 完全一致を目的とはせず、Windows 向けツールとしての実用性・可読性を優先する
- GUI テーマはライト / ダーク両対応を前提とする
- 初期テーマはダークとする
- テーマ切替 UI はメニューバーの View / 表示 配下に置く
- テーマ適用範囲はアプリ全体の主要 UI とし、Qt ウィジェット領域だけでなく波形表示エリア / Preview エリアも対象に含む
- 日本語表示と英語表示の文字長差によって、レイアウト全体が大きく崩れないことを前提とする
- GUI 層の整備に留め、既存の主要処理導線は維持する
- `main_window.py` は司令塔としての責務へ寄せ、表示構築の詳細はパネル / コンテナ側へ分散する
- 分散方針は、表示系をパネル / コンテナ単位、接続補助系を薄い補助層単位とするハイブリッド方式を採る
- 新規ファイル追加は、MS9 に必要な範囲で許容するが、将来用の過剰分解は行わない

## 5. 要件定義

### 5-1. 操作ボタン群

#### 5-1-1. 外観

- 上部操作ボタンは、アイコン＋文字の構成へ変更する
- ボタン内レイアウトは「上にアイコン、下に文字」を標準とする
- アイコンサイズ、文字位置、内側余白はボタン群内で統一する
- MS9 では、Qt 標準アイコンまたは暫定アイコンで実装する
- 外部アイコン資産への最終依存は MS9 完了条件に含めない

#### 5-1-2. サイズ統一

- ボタンサイズは、全ボタン完全共通ではなく、機能グループごとに同一サイズとする
- グループ内では幅・高さ・余白を揃える
- 高さは全グループ共通を基本とする

#### 5-1-3. 機能グルーピング

- ボタン群は機能ごとに視覚的に分離する
- グループ間の区別は、余白を基本とする
- 折り返し後は、区切り線維持を必須とせず、余白でまとまりを表現してよい
- 折り返し時も、可能な限り機能グループ単位のまとまりを維持する

#### 5-1-4. 折り返し

- ウィンドウ横幅が不足した場合、ボタン群は横スクロールではなく折り返し表示とする
- 折り返し単位は個別ボタン単位ではなく、機能グループ単位を基本とする

#### 5-1-5. 文言と省略

- 英語文言は、厳密訳よりも短く、横幅優先の UI 向け語を優先する
- 英語表記でボタン文字列が収まらない場合は、まず改行で吸収する
- 英語時の省略記号は最後の手段とし、必要な場合は Qt 標準相当の単一文字 `…` を用いる
- 日本語文言は短文化を優先し、それでも収まらない場合のみ省略を許容する

#### 5-1-6. 状態差の視覚化

- ボタンの通常 / 無効 / 押下中 / フォーカス中 などは視覚的に識別可能とする
- 未読込・未解析・処理中などの操作不可状態は、見た目で分かる表現を持たせる

#### 5-1-7. tooltip

- 全主要ボタンに tooltip を付与する
- tooltip は、ボタン文言だけでは伝わりにくい補足説明を短く示す
- tooltip 文言は、将来の文字列管理分離を前提に整理する

#### 5-1-8. 主要ボタン文言案

MS9 時点の主要ボタン文言案は以下を基準とする。  
実装時に最終調整の余地は残すが、レイアウト・省略・改行確認の一次基準として扱う。

| ボタン | 日本語案 | 英語案 |
|---|---|---|
| TEXT読込 | `TXT\n読込` | `TXT\nOpen` |
| WAV読込 | `WAV\n読込` | `WAV\nOpen` |
| 処理実行 | `処理\n実行` | `Run` |
| VMD出力 | `VMD\n保存` | `VMD\nSave` |
| Play | `再生` | `Play` |
| Stop | `停止` | `Stop` |
| Zoom In | `拡大` | `Zoom\nIn` |
| Zoom Out | `縮小` | `Zoom\nOut` |

※ `\n` は表示上の改行を意味する。

#### 5-1-9. 実装責務方針

- `OperationPanel` は表示専用寄りの責務とする
- ボタン生成・配置・文言更新・tooltip・有効 / 無効切替は `OperationPanel` 側で扱う
- 実処理接続、業務判断、処理フロー制御は `main_window.py` または補助層側で扱う

---

### 5-2. 左右レイアウトと可変分割

#### 5-2-1. 左側テキスト領域

- 左側テキスト領域は現状より狭める
- 幅の目安は、全角40文字程度が一行で読める水準とする
- ただし厳密固定値ではなく、全体レイアウトを優先した可変目安とする
- 長文は折り返し表示を前提とする
- 左側情報表示群は、`main_window.py` から独立した情報パネルへ分離する

#### 5-2-2. 右側表示領域

- 波形表示エリアと Preview エリアは、右側表示コンテナとして一体的に構成する
- `WaveformView` と `PreviewArea` の正本責務は維持しつつ、レイアウト構成と見出し・余白整理は右側コンテナ側で吸収する

#### 5-2-3. 左右分割

- テキスト領域と波形＋Preview 領域の間には、ドラッグ可能な仕切りを設ける
- 利用者はマウス操作で左右比率を変更できる
- 左右分割は専用コンテナ側で扱い、`main_window.py` に分割仕様を過度に残さない
- 右側優先で最小幅を守る方針とする

#### 5-2-4. 初期値と復元

- 左右比率の初期既定値は、左 35 : 右 65 とする
- 保存値が存在する場合は、その復元を優先する
- 分割比率の保存タイミングは、ドラッグ終了時を基本とする
- MS9 では、分割比率取得 / 適用の受け皿までを整理対象とし、保存機構の本格実装は後続でもよい

---

### 5-3. セクション見出しと情報構造

#### 5-3-1. 見出し

- 各主要領域には見出しを設ける
- 見出し表現は「テキスト見出し＋下線または区切り線」を基本とする
- 見出しは画面内の情報構造を理解しやすくする目的で用いる
- 見出しは日本語 / 英語の両対応前提とする
- 具体的な英語見出し文言は、MS9 実装時の詳細設計で最終調整してよい

#### 5-3-2. 左側情報群の余白

- 左側のラベル、入力欄、パス表示、補助情報の余白は、セクション単位で統一ルールを持たせる
- 全画面一律の余白再設計までは行わず、セクションごとの視認性改善を優先する

#### 5-3-3. 構造分散

- 左情報パネル、右表示コンテナ、左右分割コンテナの三層を基本構造とする
- `main_window.py` は、上部操作領域・中央本体・下部ステータスの束ね役へ寄せる

---

### 5-4. Preview エリア

#### 5-4-1. 行ラベル帯

- Preview の `あ / い / う / え / お` は、左側に固定ラベル帯を設けて区別する
- 行ラベル帯は、各行の意味を視覚的に理解しやすくする目的で導入する
- 固定ラベル帯はテーマ切替に追従し、周辺背景との識別性を確保する

---

### 5-5. 波形表示エリア / Preview エリアの外観

#### 5-5-1. テーマ切替

- 波形表示エリアと Preview エリアの背景テーマは、ライト / ダーク両対応を前提とする
- 初期テーマはダークとする
- テーマ切替 UI は View / 表示 メニュー配下に置く
- テーマはアプリ全体の主要 UI に適用する
- 見た目方針は、Windows 標準寄りの自然な外観を優先する
- 参照基準として既存モックアップを用いる

#### 5-5-2. 不要表記削除

- 波形表示エリアの縦軸説明 `Amp` は削除する

#### 5-5-3. GUI全体の基調色

- GUI 全体の基調色はグレー基調とする
- ダーク側の配色判断は Windows ダークモード基準を前提とする

#### 5-5-4. 背景色方針

- 波形表示エリアの背景色は、テーマに応じて切り替える
- ダークテーマ側では、暗いグレー系背景を基準とする
- ライトテーマ側の具体色は、全体整合と可読性を優先して決定する

#### 5-5-5. 配色再設計範囲

- 再配色対象は、背景・波形本体・軸・補助線・再生カーソルに加え、選択状態や強調色も含める
- 一部要素だけではなく、表示系全体の統一感を確保する
- Qt 側配色と matplotlib / Preview 側配色が大きく乖離しないことを重視する

#### 5-5-6. 波形本体の色方針

- 波形本体は寒色系単色を基本候補とする
- ただし、最終判断は背景とのコントラストを最優先とする
- 具体的な色コードは本書では固定せず、方向性ベースで決定する

#### 5-5-7. 再生カーソルの色方針

- 再生カーソルは赤系など、直感的に分かりやすい色を用いる
- 波形本体とは明確に識別できることを前提とする

#### 5-5-8. 補助線・区切り線・軸線

- 補助線・区切り線・軸線は、波形本体より弱く扱う
- ただし、区切りの認識に必要な線は、視認できる水準ではっきり出す

#### 5-5-9. 波形本体と再生カーソルの役割分離

- 波形本体の色と再生カーソルの色は、明確に役割分離する
- 状態変化の主表現を波形本体へ過度に持たせず、再生位置や強調要素は別系統で扱う

---

### 5-6. ステータスバー

#### 5-6-1. 基本構成

- ステータスバーは、少なくとも「状態表示」と「メッセージ表示」の2領域構成とする
- 情報量を過剰に増やさず、現在状態と補助メッセージの役割を分離する

#### 5-6-2. 状態表示の方針

- 状態表示は、成功 / 失敗 / 注意 の3区分を優先する
- 状態表示文言は、将来の英語化を見据えて短い状態語を前提に設計する

#### 5-6-3. メッセージ表示の方針

- メッセージ表示は、エラー / 警告内容を優先して表示する
- 通常時は短い補助文を表示する
- 異常時は通常補助文を上書きしてよい

#### 5-6-4. エラー / 警告時の挙動

- エラー / 警告時は、一時的に通常状態表示を上書きする
- MS9 時点では、状態欄とメッセージ欄の個別文言確定までは行わず、状態種別方針の固定に留める

#### 5-6-5. 再生中表示

- 再生中の情報は、ステータスバー上では「再生中であること」の表示に留める
- フレーム値や時刻値は、MS9 のステータスバー常時表示対象には含めない

#### 5-6-6. 実装責務方針

- `StatusPanel` は表示専用に近いが、軽い整形責務を持ってよい
- 外側は状態種別とメッセージを渡し、色・強調・表示形式の切替は `StatusPanel` 側で吸収する
- 業務判断や状態遷移の判定は `main_window.py` または外側ロジック側で扱う

---

### 5-7. 分析終了通知

- 分析処理終了時には、成功・失敗のいずれでも通知音を鳴らす
- 通知音は Windows 標準音を用いる
- 成功 / 失敗で音種は分けず、同一の Windows 標準音を用いる
- 音種は、警告色の強すぎない情報通知寄りの穏やかな音を優先する

---

### 5-8. 多言語化を見据えた前提

- 日本語 / 英語で表示文字列長が変化しても、ボタンや見出しのレイアウトが大きく崩れないことを前提とする
- 英語 UI ラベルは、厳密訳よりも短く、横幅優先の分かりやすい操作語を優先する
- 文字列長差への耐性は MS9 の GUI 設計側で先行して整える
- 実際の文字列定義集約と切替機構そのものは後続マイルストーンで扱う
- ただし、MS9 の時点で GUI 文字列を後続で集約しやすい構造へ寄せる
- `main_window.py` への文字列ベタ書き集中は、可能な範囲で緩和する

---

### 5-9. ウィンドウサイズ制約

- 最小ウィンドウサイズは、ボタンが2段折り返しになった状態でも主要操作が破綻しないことを基準に設定する
- 極端な縮小で波形表示、Preview、左側入力領域が実用不能にならないことを重視する
- 右側表示領域の可視性を優先し、左右分割時の最小幅制約を設計する

---

### 5-10. GUI構造分散方針

#### 5-10-1. 分散の基本方針

- `main_window.py` の責務肥大化を抑えるため、GUI 構築詳細はパネル / コンテナへ分散する
- 分散はハイブリッド方式とし、表示系と接続補助系を分けて整理する

#### 5-10-2. 表示系の分散先

- 上部操作領域は `OperationPanel` を主担当とする
- 左情報表示群は独立パネルを新設して受ける
- 右側表示領域は波形 + Preview を束ねる独立コンテナを新設して受ける
- 左右可変分割は専用コンテナを新設して受ける
- 下部状態表示は `StatusPanel` を主担当とする

#### 5-10-3. `main_window.py` に残す責務

- 各主要パネル / コンテナの束ね
- メニュー構築の起点
- ボタン押下と処理実行導線の接続
- 再生制御接続の起点
- viewport 同期接続の起点
- 全体状態更新の司令塔責務

#### 5-10-4. 補助層

- 接続補助、レイアウト補助、文字列管理、設定受け皿などは、必要な範囲で薄い補助層を追加してよい
- ただし、MS9 の段階では過剰な抽象化や将来専用の大規模基盤化は行わない

---

## 6. 後続マイルストーンとの関係

- 従来の MS8E（多言語化・設定永続化）は、MS9 新設に伴い MS10 として扱う
- 従来の MS8F（出力品質拡張）は、MS11 として扱う
- MS9 は、MS10 実装前に GUI 基盤を整えるための前段マイルストーンと位置付ける
- MS9 では、MS10 で本格化するテーマ保存・分割比率保存・文字列集約に備えた GUI 側受け皿までを整える

---

## 7. 完了条件

MS9 は、以下を満たした時点で完了とする。

- 上部操作ボタンがアイコン＋文字化されている
- ボタンが機能グループごとに同一サイズで整理されている
- ボタン群がグループ単位で折り返し表示できる
- 英語表記時の文字省略が必要時のみ `…` で処理される
- ボタン内レイアウトが「上にアイコン、下に文字」で統一されている
- 全主要ボタンに tooltip が付与されている
- 左右領域がドラッグでサイズ変更可能である
- 左右比率の既定値が左35 : 右65 として成立する
