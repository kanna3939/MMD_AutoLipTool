- Project: MMD_AutoLipTool

## 1. 目的

MS11-3 の目的は、`writer.py` における区間形状生成を、MS11-2 の **4点・単一上辺台形** から、**複数上辺点を持つ多点エンベロープ** へ拡張することである。

主対象は **同一母音の近接イベント群** とし、それらを 1 つの連続形状として出力できるようにする。  
その際、MS11-1 / MS11-2 / FIX01 / FIX02 で整備済みの以下を壊さないことを前提とする。

- 重複統合
- short open / short pulse 抑制
- 許容外非ゼロ除去
- 不要ゼロ prune
- 形状保護
- VMD 構造正常性

- **同一母音の近接イベント群** を主対象とする

- 同一エンベロープ内では、谷があっても **原則として完全閉口までは落とさない**

- 各イベントの `peak_value` を基本採用
- 必要最小限の平滑化のみ許容

### 4-7. 保護単位
- 多点エンベロープ全体を基本保護する
- ただし明らかな破綻部のみ後段で整理可能とする

### 4-8. フォールバック順
- まず MS11-2 単一上辺台形へフォールバック
- それも無理な場合のみ legacy fallback へ落とす

## 6. 問題定義

MS11-2 では、1 イベントを **4点・単一上辺台形** で表現できるようになった。  
しかし現状では、同一母音が近接して連続する場合でも、各イベントは原則として個別 shape として出力される。

その結果、以下の課題が残る。

1. 同一母音連続時に、形状が分断されやすい
2. 1 区間内で上辺の高さ変化を表しにくい
3. 長めの発話や近接連続発話を、より自然な 1 本の口形として出しにくい
4. MS11-2 の単一上辺では、複数のピークを持つ同一母音形状を表現できない

---

## 7. 基本方針

### 7-1. 基本思想

MS11-3 では、**同一母音の近接イベント群を 1 つの多点エンベロープとして扱う**。

ただし、現行の timeline 構造を大きく変えず、まずは writer 側で group 化し、そこから多点 shape を構築する。

### 7-2. 形状の考え方

- グループ先頭に開始ゼロ
- グループ末尾に終了ゼロ
- グループ内の各イベントに対応する上辺点を持つ
- 上辺点の間には谷を持てる
- ただし谷は原則として 0.0 まで閉じない

### 7-3. 段階的フォールバック

多点 shape が frame ベースで成立しない場合は、以下の順で段階的に落とす。

1. MS11-3 多点エンベロープ
2. MS11-2 単一上辺台形
3. legacy fallback

## 8. 想定データ構造

MS11-2 の `AsymmetricTrapezoidSpec` は維持しつつ、MS11-3 用に多点形状の内部仕様を追加する。

### 8-1. 追加候補

#### `EnvelopeControlPoint`

#### `MultiPointEnvelopeSpec`

- 最終的には `MorphFrame` 列へ展開可能である
- 保護導線に必要なメタ情報を保持できる

## 9. 実装方針詳細

### 9-1. グルーピング

同一母音イベント列について、以下を満たすものを 1 グループに束ねる。

- 母音が同じ
- イベント間ギャップが近い
- 秒ベースでは連続性がある
- frame 化後にも shape として成立可能

近接ギャップ上限は、初期実装では **4 frame 程度** を基準とする。

### 9-2. 多点エンベロープ生成

各グループについて、以下を行う。

1. グループ全体の開始 / 終了 frame を決定する
2. 各イベントの `time_sec` / `peak_value` から上辺点候補を作る
3. 上辺点の間に谷を配置する
4. 谷は 0.0 に落とさず、同一母音連続として成立する最小開口を持たせる
5. 必要最小限の高さ平滑化を適用する

### 9-3. frame 成立判定

生成した多点 shape について、frame 化後に以下を確認する。

- 点順序が破綻していないか
- 開始ゼロ / 終了ゼロが成立するか
- 上辺点が潰れすぎていないか
- shape 全体として意味のある連続形状になっているか

不成立時は、MS11-2 形状へフォールバックする。

### 9-4. 保護拡張

MS11-3 shape は、MS11-2 よりも広い単位で保護する。

- 保護単位は「主要点」ではなく **多点エンベロープ全体**
- short open / short pulse 抑制が、谷や肩を不必要に潰さないようにする
- ただし、丸め後に明らかに破綻した部分だけは整理可能とする

### 9-5. FIX01 / FIX02 意図の維持

MS11-3 導入後も、以下は維持する。

- 許容外非ゼロ除去
- 必要ゼロ維持
- 不要ゼロ prune
- 発話イベント内部の shape 保護
- ゼロ縮退の再発防止
