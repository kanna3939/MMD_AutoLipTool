# MS10_Implementation_Plan

## 1. 目的

MS10 は、現行 GUI 基盤の上に **設定永続化** と **多言語化本実装** を導入するマイルストーンとする。  
MS9 / MS9-2 / GUIFIX06 までで整備した GUI・表示層を前提に、以下を成立させることを目的とする。

- 日本語 / 英語の実行中切替
- 言語設定の次回起動時復元
- 履歴の次回起動時復元
- 重要 UI 設定の永続化
- モーフ最大値の永続化
- 文字列定義の正本一元化
- 設定破損時・保存失敗時の安全動作

## 3. 設定永続化方針

### 3-1. 保存対象

MS10 で正式に永続化対象とするのは、次の構成とする。

#### 重要設定（都度保存）
- テーマ設定
- splitter 比率
- ウィンドウサイズ
- 言語設定
- モーフ最大値

#### 終了時保存対象
- 履歴の各項目
  - 最近使った TEXT ファイル履歴
  - 最近使った WAV ファイル履歴

### 3-2. 保存対象外

次の項目は MS10 では永続化しない。

- 波形表示オプション
  - フレームグリッド ON/OFF
  - 母音ラベル ON/OFF
  - イベント区間表示 ON/OFF
- その他、一時的表示状態
- 再生状態
- 再生位置
- 一時選択状態
- Viewport / Zoom / Pan の内部状態
- 描画内部状態

### 3-3. 保存形式

- 設定ファイル形式は `.ini`
- 新規作成ファイル名は **`MMD_AutoLipTool.ini`**

### 3-4. 保存先

- 保存先は **exe ルート**
- 開発実行時も同じ方針に準拠し、アプリ実行基準位置から扱う

### 3-5. 保存タイミング

#### 即時保存
- テーマ変更時
- 言語変更時
- モーフ最大値変更時

#### 操作確定寄りで保存
- splitter 比率
- ウィンドウサイズ

#### 終了時保存
- 最近使った TEXT ファイル履歴
- 最近使った WAV ファイル履歴

### 3-6. モーフ最大値の扱い

- モーフ最大値は **表示専用の一時状態ではなく、出力結果に影響する利用者設定** として扱う
- そのため、MS10 では重要設定として永続化対象に含める
- 永続化は既存の処理仕様を変更するものではなく、既存入力値の復元を目的とする

## 4. 起動時復元方針

起動時の設定復元は、次の順序で適用する。

1. ウィンドウサイズ
2. splitter
3. 言語
4. UI 構築
5. 文言反映
6. テーマ
7. モーフ最大値
8. 履歴復元

この順序を正とし、実装中に変更しない。

### 4-1. 画面サイズ不整合時

- 保存済みウィンドウサイズが現在の画面環境に収まらない場合は、**現在画面内に収まる安全サイズへ補正する**
- ただし、最小サイズ制約は維持する
- 単純に保存値を優先して画面外へはみ出させない
- 画面不整合時に直ちに既定サイズへ全戻しするのではなく、まずは安全な範囲で補正する

### 4-2. 復元時シグナルへの配慮

- 復元時に UI 値変更シグナルが不要に副作用を起こさないよう配慮する
- 特にモーフ最大値の復元時は、必要に応じてシグナル抑制または初期化中フラグで制御する

## 5. 多言語化方針

### 5-1. 対応言語

- 日本語
- 英語

### 5-2. 言語切替 UI

- **メニュー方式** とする
- **View** 配下に **Language** サブメニューを置く
- 項目名は固定で **`日本語` / `English`** とする
- 現在表示言語に応じて項目名自体を切り替える方式は採らない

### 5-3. 初期言語決定ルール

初回起動時、または保存済み言語設定が取得できない場合は、OS 言語設定を読む。  
読めない場合の優先順は次の通りとする。

1. 英語
2. 日本語

### 5-4. 多言語化対象範囲

MS10 では、次を一括して多言語化対象とする。

- メニュー
- ファイルダイアログ
- 警告文
- 状態文
- 空状態文言
- 軸ラベル
- ツールチップ
- バージョン情報
- ボタン
- ラベル
- セクション見出し

対象範囲から意図的に除外する文言は設けない。  
ソース上の直書き文字列は原則として回収対象とする。

### 5-5. 英語文言方針

- **折衷型** とする
- ボタン・メニューは短めの表記を優先する
- ダイアログ・警告文・説明文は自然な英文を優先する
- 英語文言は、画面幅都合だけで過度に不自然な省略へ寄せない

### 5-6. ステータス表示方針

- 再生中ステータスは **固定文言のみ** とする
- 再生中に現在フレーム値を付与する仕様は MS10 では採用しない

## 6. `i18n_strings.py` 運用ルール

### 6-1. 基本方針

- `i18n_strings.py` を文字列定義の正本とする
- 多言語文字列は外部 JSON へ出さない
- Python 内部管理を継続する

### 6-2. 構成方針

- **画面責務ごとのクラス分割** を維持する
- 各クラス内は **用途別の大文字定数名** で統一する

例:
- `MainWindowStrings`
- `OperationPanelStrings`
- `StatusPanelStrings`
- `WaveformViewStrings`
- `CommonStrings`

### 6-3. 共通文言

- OK / Cancel / Error など、複数箇所で共通利用する文言は共通領域へ集約する

### 6-4. フォーマット文言

- フォーマット文言はテンプレート文字列として定義する
- 埋め込み値の組み立ては呼び出し側で行う

### 6-5. 文言作成方針

- 文言の厳密固定は行わない
- 一般的に理解できる内容であれば、実装時の文言は AI 判断に委ねる

### 6-6. バージョン情報の扱い

- **バージョン値そのもの** は `i18n_strings.py` の固定文字列として持たない
- バージョン値の正本は **`pyproject.toml` 等のアプリ版数定義側** に置く
- `i18n_strings.py` 側では、必要に応じて表示テンプレートのみを持つ
- UI 表示時に版数定義から文言を組み立てる

## 8. 安全動作方針

### 8-1. 設定破損時

- 設定ファイルが壊れていても起動不能にしない
- 破損時は安全なデフォルト値で起動する
- 不正値は既定値へフォールバックする
- 破損した `.ini` は **そのまま残す**
- 起動時は破損内容を利用しない
- 以後の保存時に通常の正常設定として上書きしてよい

### 8-2. 保存失敗時

保存失敗時は、次の方針とする。

- **初回失敗時のみ** 警告ダイアログを 1 回表示する
- 以後、そのセッション中の保存処理は無効化する
- 画面上の変更はそのセッション中は反映してよい
- ただし、次回起動時には保持されないことを利用者へ伝える
- 詳細な例外全文は既定表示しない
- 文言は要約中心とする

---

## 9. 履歴管理方針

- 永続化後も、現行と同じ件数上限を維持する
- 無効な履歴項目は起動時に除去する
- 履歴保存は終了時に行う
- 履歴保存失敗時もアプリ本体機能は継続利用可能とする
- TEXT 履歴と WAV 履歴は混線させない

### 9-1. 履歴 0 件時の表示

- 履歴が 0 件のときは、空メニューにはしない
- プレースホルダ 1 項目を表示する
- プレースホルダは非活性表示とする
- 文言例は `履歴なし` / `No recent files` を基準とし、最終文言は実装時に一般的な範囲で調整可とする

## 13. 実装上の基本ルール

- 既存の GUI 責務分離を崩さない
- 既存の出力仕様・解析仕様を変更しない
- 多言語化は文字列差し替えに限定し、UI 再設計へ広げない
- 永続化は定義済み対象に留める
- 新規文言は一般的に理解できる範囲で AI 判断可
- 仕様未確定事項を増やさない
