今回の主目的は、`writer.py` 側の正規化・保護・cleanup に起因する内容不整合を修正し、MS11-2 の基本仕様を安定化させることであった。

## 2. 今回未対応とした課題

### 課題1：無音に見える区間で開口キーが残るケースがある

#### 概要
生成結果の一部において、ユーザーの聴感上は無音に見える区間にもかかわらず、母音モーフの開口キーが残るケースがある。

代表例として、`お` の 419〜427 frame 付近で、本来は無音部分とみなしたい区間に開口キーが存在することが確認された。

#### 現時点での見立て
現時点の調査では、この問題は `writer.py` 単独の不整合というより、`pipeline.py` 側で当該区間に有効な母音イベントが割り当てられていることが主因と考えられる。

つまり、

- テキスト由来の母音イベント割当て
- 区間境界調整
- RMS ベースの peak_value 付与
- 最小区間長の確保

などの前段処理により、ユーザー視点では無音に感じる区間でも、内部的には有効な母音イベントとして残っている可能性が高い。

その結果、`writer.py` はそれを正当なイベントとして扱い、許容範囲内のキーとして VMD に出力していると考えられる。

#### 今回未対応とした理由
この問題は、MS11-2_FIX01 / FIX02 で扱った `writer.py` 後段の局所不整合とは性質が異なる。

今回の課題は主に、

- どの低エネルギー区間を無音とみなすか
- 短い母音イベントを残すか落とすか
- 母音イベントの境界と存在判定をどの基準で決めるか

といった、`pipeline.py` 側のイベント生成方針・品質調整ポリシーに近い問題である。

このため、MS11-2 の修正スコープに含めると、
`writer.py` の安定化修正ではなく、前段のタイミング生成仕様の見直しへ発展しやすく、影響範囲が広がる。

以上より、本件は **MS11-2 の修正完了条件には含めず、将来の品質改善課題として切り分ける** 判断とした。

---

## 3. 現時点での結論

MS11-2 の修正対象としていた以下は完了とみなす。

- MS11-2 の 4点非対称・単一上辺台形仕様の維持
- FIX01 による不要ゼロ連続・無音区間非ゼロの主要改善
- FIX02 による発話区間ゼロ縮退の主要改善
- VMD 構造正常性の維持
- writer 系テストおよび追加テストによる回帰確認

一方で、**無音に見える区間で開口キーが残るケース** は、現時点では `pipeline.py` 側のイベント生成品質に近い既知課題として残す。

---

## 4. 将来対応時の観点

将来この課題に対応する場合は、`writer.py` の局所修正ではなく、主に以下の観点で検討する。

- `pipeline.py` における母音イベント存在判定の妥当性
- 低エネルギー区間を無音として落とす閾値または条件
- 近接イベントや短区間イベントの扱い
- テキスト由来の母音割当てと音声実エネルギーの整合性
- 最終的な VMD 品質を優先したイベント剪定ルール

ただし、これらは MS11-2 の修正範囲外であり、別マイルストーンまたは別課題として扱う。
