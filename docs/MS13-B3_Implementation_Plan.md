## 0. 文書目的

本書は、`MS13-B3: メニュー / 基本操作列 / ステータスの最小配置` の実装方針を固定するための文書である。

本 block の役割は、`MS13-B2` で成立したメインフレーム最小骨格の上に、

- 最小のメニューバーを配置する
- 上部操作域へ主要操作の最小配置を行う
- 下部ステータス域へ最小の表示導線を置く
- 後続 block や `MS14` で実機能を接続しやすい受け口を整える

ことである。

本書は詳細コード全文を規定するものではない。  
本書は、**Antigravity が MS13-B3 を scope を崩さず実装するための実装計画書**として扱う。

## 2. 本 block で固定する前提

本書は、以下の判断結果を前提とする。

### 2.1 メニュー粒度
- 現行アプリに対応する**主要メニュー名だけ**を並べる
- 未実装項目は無効化する
- 現行機能 parity の完成までは目指さない

### 2.2 基本操作列
- 上部操作域には主要ボタンを配置する
- ただし、未接続または最小接続に留める
- 実処理本体や file dialog 本体はまだ実装しない

### 2.3 ステータス域
- 単一ラベルだけで終わらせず、将来拡張しやすい**最小コンテナ構成**を採る
- ただし、進捗表示や複数状態表示まではまだ入れない

### 2.4 未実装操作の見せ方
- 未実装のメニュー項目やボタンは**表示するが disabled にする**
- 「未実装」ダイアログ導線はこの block では作らない

### 2.5 実装範囲
- 見た目の配置に加え、将来接続しやすい**イベント受け口だけ**は作ってよい
- 実機能接続は `MS14` 以降へ送る

### 2.6 上部操作列に含める操作範囲
- `TEXT選択 / WAV選択 / 解析実行 / VMD保存`
- 上記に加えて、**再生系主要ボタンの置き場**も含める
- ただし、再生機能自体は未実装とする

## 3. MS13-B3 の目的

MS13-B3 の目的は、以下の 3 点である。

### 3.1 最小 UI 導線の可視化
アプリ起動直後に、ユーザーが「何をするツールか」を把握できる最小 UI を置く。

### 3.2 今後の機能接続先の明確化
後続 block や `MS14` 以降で、

- file open
- process
- save
- playback
- status update

を接続しやすい構造を先に整える。

### 3.3 骨格から最低限のアプリらしさへ進める
B2 の placeholder 骨格から一歩進めて、  
**まだ未完成でも、操作単位のまとまりが見える状態**を作る。

## 7. 実装方針

## 7.1 B2 の骨格の上に、最小 UI 群を載せる
B3 では、新しい領域分割は増やさない。  
B2 で作成した

- 上部操作域
- 中央主領域
- 下部ステータス域

のうち、主に上部と下部へ最小 UI を配置する。

### 7.2 主要メニュー名だけを先に固定する
メニューは「動作完成」よりも「構造固定」を優先する。  
主要メニュー名だけを先に置き、未実装項目は disabled とする。

### 7.3 基本操作列は見えるが未完成でよい
上部操作列は、将来の主導線をユーザーと実装者の双方に見せるために配置する。  
ただし、今は見えるだけで十分な操作もある。

### 7.4 ステータス域は将来拡張しやすい最小形にする
単一ラベルだけで終わらせるのではなく、  
後で progress や warning 表示が入っても崩れにくい最小構造を採る。

### 7.5 実機能は先回りしない
B3 の本質は**最小配置**であり、機能復旧ではない。  
実機能に入りたくなっても、`MS14` または後続 block へ送る。

## 13. 想定リスク

### 13.1 B3 に MS14 の内容が流入すること
ボタンを置いた結果、そのまま file open や save 実装まで進めたくなる可能性が高い。  
今回は受け口または disabled 表現までに留める。

### 13.2 再生系を作り込みたくなること
再生ボタンの置き場を作ると、実再生やタイマー導線へ入りたくなる。  
それは scope 外とする。

### 13.3 ステータス域を先に作り込みたくなること
progress / warning / busy を先に入れると B4 や後続 block と混ざる。  
今回は最小ラベル + 最小コンテナに留める。

### 13.4 メニューを作り込みすぎること
詳細なメニュー構成や submenu を増やしすぎると、実機能 parity へ寄って scope が膨らむ。  
主要メニュー名の可視化を優先する。

## 15. 要約

MS13-B3 は、`MS13-B2` で作成した最小骨格の上に、

- 主要メニュー名を持つ最小メニューバーを配置し
- 主要ボタンを持つ最小操作列を上部操作域へ配置し
- 最小のステータス表示導線を下部ステータス域へ配置する

block である。

この block では、

- 未実装項目は disabled とする
- 実機能接続は最小に留める
- file open / save / replay / settings などの本格接続は行わない

という境界を守る。

本 block の本質は、  
**「まだ未完成でも、アプリの主要操作と状態表示の骨格が見える状態」を作ること**  
である。

本書を `MS13-B3` の実装計画書として固定し、  
Antigravity は本書の範囲内で block 実装を進めるものとする。
